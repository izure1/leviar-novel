# 💾 세이브와 로드 (Save & Load)

본 문서는 시나리오 작성자가 세이브 기능이 어떻게 작동하는지, 그리고 데이터를 다룰 때 어떤 점을 주의해야 하는지 안내합니다. (Explanation & How-to)

---

## 1. 개요 (Overview)

Fumika 엔진은 플레이어가 게임을 저장(`novel.save()`)할 때, **현재 화면에 보이는 모든 것과 진행 상황을 자동으로 기록**합니다.  
시나리오 작성자는 엔진이 어떤 데이터를 보존하는지 이해함으로써, 더욱 안전하고 버그 없는 게임을 제작할 수 있습니다.

### 엔진이 자동으로 저장하는 것들
*   **진행 위치**: 현재 어느 씬(`Scene`)의 몇 번째 대사를 읽고 있었는지.
*   **변수 데이터**: 플레이어의 호감도, 소지품, 선택지 결과 등 (`set` 예약어로 저장한 데이터).
*   **화면 및 소리**: 현재 띄워진 배경, 캐릭터 이미지, 재생 중인 배경음악(BGM)과 그 재생 시점.
*   **씬 중첩(Call) 스택**: `call` 예약어를 통해 서브씬으로 진입했을 경우, 원래 씬으로 돌아가기 위한 모든 기억들.

---

## 2. 변수(데이터) 저장 시 주의사항

세이브 기능이 완벽하게 작동하려면, `set` 예약어로 변수에 데이터를 넣을 때 **반드시 지켜야 할 규칙**이 있습니다.  

### ❌ 절대 저장하면 안 되는 데이터 (에러 발생 원인)
세이브 파일은 최종적으로 텍스트(JSON) 형태로 변환되어 기기에 저장됩니다. 따라서 텍스트로 변환할 수 없는 복잡한 시스템 데이터는 변수에 담으면 안 됩니다.

*   **화면 요소 (DOM Element)**: `document.getElementById('my-div')` 같은 HTML 요소.
*   **함수 (Function)**: 실행 가능한 코드 블록 자체.
*   **서로 참조하는 데이터 (Circular Reference)**: A가 B를 포함하고, B가 다시 A를 포함하는 무한 반복 데이터.

> [!WARNING]
> 위와 같은 데이터를 변수에 넣으면 플레이어가 게임을 저장할 때 화면이 멈추거나 **치명적인 세이브 에러**가 발생합니다!

### ✅ 올바른 데이터 형식 (권장)
*   **숫자 (Number)**: `1`, `100`, `3.14`
*   **문자 (String)**: `'sword'`, `'happy'`
*   **참/거짓 (Boolean)**: `true`, `false`
*   **간단한 배열/객체**: `[1, 2, 3]`, `{ item: 'potion', count: 2 }`

---

## 3. 전역 변수와 지역 변수의 세이브 차이

*   **전역 변수 (일반 변수명)**: 게임이 끝날 때까지 영원히 세이브 데이터에 남아있습니다. (예: `set('gold', 100)`)
*   **지역 변수 (씬의 `variables` 블록에서 `_`로 시작하여 선언된 변수명)**: 현재 씬을 진행 중일 때만 유지되며, **다른 씬으로 완전히 넘어가면 자동으로 소거**되어 메모리를 최적화합니다. (예: `set('_door_click', 1)`)

> [!TIP]
> 씬 안에서만 잠깐 쓰고 버릴 데이터는 꼭 `_` (언더스코어)를 붙여 지역 변수로 만드세요! 세이브 파일 용량이 가벼워지고 엉뚱한 씬에서 데이터가 꼬이는 버그를 막을 수 있습니다.
