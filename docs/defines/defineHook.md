# ⚓ defineHook (훅 정의 헬퍼)

본 문서는 `defineHook` 헬퍼 함수를 사용하여 특정 장면(Scene) 내에서 발생하는 모듈 및 엔진 이벤트를 가로채고 제어하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`defineHook`은 씬 스코프 내에서 유효한 이벤트 인터셉터를 체계적으로 정의하기 위한 헬퍼 함수입니다.  
이 함수를 통해 등록된 훅은 씬이 활성화될 때 자동으로 구독을 시작하며, 씬이 종료되거나 전환될 때 엔진에 의해 안전하게 해제됩니다.  
개발자가 수동으로 이벤트 리스너를 해제할 필요가 없으므로, 메모리 누수를 방지하고 코드의 무결성을 유지하는 데 큰 이점을 제공합니다.  

| 핵심 항목 | 상세 내용 |
| :--- | :--- |
| **목적** | 씬 전용 이벤트 리스너 등록 및 생명주기 자동 관리 |
| **대상** | `novel.config.ts`에 등록된 모듈 전용 훅 및 `novel:` 엔진 코어 훅 |
| **이점** | 자동 구독 해제를 통한 메모리 누수 방지 및 씬 단위 로직 격리 |

*   **[상세 가이드: Hooks 아키텍처 개요](../concepts.md#hooks)**: 훅 시스템의 데이터 릴레이 방식에 대한 전반적인 이해를 돕습니다.  

---

## 2. 사용 방법 (Usage)

`defineHook`은 현재의 `config` 인스턴스를 주입받아, 훅 식별자와 콜백 객체가 매핑된 훅 맵(Hook Map)을 정의하는 커리 형식을 취합니다.  

### 2.1. 시나리오 연동 예제
장면 내에서 대사 출력 시점을 가로채거나 엔진의 흐름 제어 이벤트에 개입하는 구현 예시입니다.  

```ts
// scene-forest.ts
import config from '../novel.config'
import { defineScene, defineHook } from 'fumika'

export default defineScene({
  config,
  // 씬 생명주기에 따라 훅이 자동으로 관리됩니다.  
  hooks: defineHook(config)({
    // 1. 특정 모듈에서 발생하는 훅 구독
    'dialogue:text-run': {
      onBefore: (state) => {
        console.log(`[시스템] 현재 처리 중인 대사: ${state.text}`);
        return state // 보정된 데이터를 반환하여 릴레이합니다.  
      }
    },
    // 2. 엔진 코어에서 발생하는 전역 훅 구독
    'novel:next': {
      onceAfter: (value) => {
        console.log('[시스템] 다음 시나리오 단계로 이동이 완료되었습니다.');
        return value
      }
    }
  })
}, () => [ ... ])
```

---

## 3. 라이프사이클 메서드 (Methods)

각 훅 식별자 하위에서 다음의 실행 시점 메서드를 정의할 수 있습니다.  

| 메서드 명칭 | 상세 설명 |
| :--- | :--- |
| **`onBefore`** | 핵심 로직 실행 직전에 호출됩니다.  데이터를 인터셉트하여 수정하거나 흐름을 제어할 수 있습니다. |
| **`onAfter`** | 로직 실행이 완료된 후에 호출됩니다.  최종적으로 결정된 결과값을 확인할 때 활용합니다. |
| **`onceBefore`** | `onBefore`와 동일하나, 씬 실행 중 단 한 번만 호출되도록 보장합니다. |
| **`onceAfter`** | `onAfter`와 동일하나, 단 한 번의 실행 후 구독이 자동 중단됩니다. |

---

## 4. 작동 메커니즘 (Internal Logic)

1.  **자동 등록 파이프라인**: `defineScene`에 전달된 `hooks` 정의는 엔진이 해당 씬을 활성화하는 즉시 `_register()` 메서드를 통해 시스템에 통합됩니다.  
2.  **안전한 자원 회수**: 씬 전환 시 엔진은 `_unregister()`를 호출하여 해당 씬에 종속된 모든 훅 리스너를 소거함으로써 런타임 오버헤드를 방지합니다.  
3.  **스코프 격리 정책**: `defineHook`으로 선언된 로직은 해당 씬의 실행 컨텍스트 내부에서만 유효하므로, 다른 씬의 시나리오 진행에 영향을 주지 않는 안정적인 로직 격리를 보장합니다.  

---

## 5. 주의 사항 (Edge Cases)

*   **다중 리스너 우선순위**: 동일한 훅 식별자에 대해 여러 개의 콜백을 등록할 수 있으며, 코드 상에 정의된 순서에 따라 순차적으로 실행됩니다.  
*   **데이터 무결성 유지**: `onBefore` 메서드에서 결과값을 반환할 때는 반드시 전달받은 `state` 객체와 동일한 스키마를 유지해야 엔진의 다음 연산이 정상적으로 수행됩니다.  
*   **코어 훅 식별 규칙**: 엔진 자체에서 발생하는 시스템 훅(예: `next`, `scene-load`)을 구독할 때는 식별의 명확성을 위해 반드시 `novel:` 접두사를 사용해 주시기 바랍니다.
