# 01. 환경 구축 및 설치 (Installation) 🛠️

`fumika` 엔진을 사용하여 게임을 만들기 위한 첫 번째 단계입니다. 단순히 라이브러리를 설치하는 것을 넘어, 왜 이런 환경이 필요한지 이해하는 것이 중요합니다.

---

## 프로젝트 생성 및 초기화

비어 있는 새로운 폴더를 만들고 프로젝트를 초기화합니다. `fumika`는 최신 웹 표준을 따르므로, 모든 자산과 코드를 체계적으로 관리할 준비를 합니다.

```bash
mkdir my-novel-game
cd my-novel-game
npm init -y
```

## 엔진 설치

`npm`을 통해 `fumika` 엔진을 설치합니다. 이 엔진은 복잡한 그래픽 렌더링과 시나리오 흐름 제어를 대신 처리해 주는 핵심 도구입니다.

```bash
npm install fumika
```

## 💡 깊이 알아보기: 왜 TypeScript와 VS Code를 쓰나요?

`fumika`는 개발자의 실수를 사전에 방지하기 위해 **강력한 타입 시스템**을 제공합니다. 

**예를 들어**, 시나리오를 작성하다가 캐릭터 이름을 `aris`라고 써야 하는데 실수로 `ariss`라고 오타를 냈다고 가정해 봅시다. 일반적인 자바스크립트 환경에서는 게임을 실행해 보기 전까지는 이 에러를 알 수 없지만, TypeScript 환경의 VS Code에서는 즉시 빨간 줄이 그어지며 수정을 제안합니다.

## 권장 환경 및 도구
*   **Node.js**: v18 이상 (엔진의 최신 기능을 활용하기 위해 필요합니다)
*   **TypeScript**: v5.0 이상 (강력한 타입 추론과 자동완성을 위해 필수적입니다)
*   **VS Code**: `fumika` 엔진의 모든 기능은 VS Code의 인텔리센스(IntelliSense) 기능을 통해 최고의 효율을 발휘합니다.

## 추천하는 프로젝트 구조

처음에는 아래와 같이 최소한의 파일로 시작하는 것을 권장합니다.

*   `/assets`: 이미지, 사운드 등 원본 리소스 보관
*   `novel.config.ts`: 게임의 전역 설정 파일
*   `main.ts`: 엔진을 실행하는 진입점
