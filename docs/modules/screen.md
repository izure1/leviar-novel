# 🎬 Screen Module

---

## 1. 개요 (Overview)

`screen` 모듈은 화면 전체를 대상으로 하는 전환 효과 및 특수 연출을 담당합니다. 부드럽게 화면을 덮거나 드러내는 **페이드(Fade)**, 순간적으로 강한 빛을 내는 **플래시(Flash)**, 그리고 방향성이 있는 화면 전환인 **와이프(Wipe)** 기능을 제공합니다.

---

## 2. 주요 커맨드 (Commands)

### 2.1. 화면 페이드 (`screen-fade`)

화면 전체를 특정 색상으로 덮거나(`out`), 덮인 상태에서 다시 나타나게(`in`) 합니다.

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `dir` | `'in' \| 'out'` | ✅ | 페이드 방향입니다. (`in`: 화면 나타남, `out`: 화면 덮임) |
| `preset` | `FadePreset` | - | 색상 프리셋 (`black`, `white`, `red`, `dream`, `sepia`) |
| `duration` | `number` | - | 애니메이션 지속 시간(ms) |
| `disable` | `boolean` | - | 효과 진행 중 사용자 입력을 차단할지 여부 |

### 2.2. 화면 플래시 (`screen-flash`)

화면을 순간적으로 번쩍이게 하여 충격이나 카메라 셔터 등의 효과를 연출합니다.

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `preset` | `FlashPreset` | - | 플래시 색상 프리셋 (`white`, `red`, `yellow`) |
| `duration` | `number` | - | 한 번의 플래시 지속 시간(ms) |
| `repeat` | `number` | - | 반복 횟수 |

### 2.3. 화면 와이프 (`screen-wipe`)

그라데이션 와이프 효과를 사용하여 방향성이 있는 화면 전환을 구현합니다.

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `dir` | `'in' \| 'out'` | ✅ | 와이프 방향입니다. (`in`: 화면 나타남, `out`: 화면 덮임) |
| `preset` | `WipePreset` | - | 진행 방향 프리셋 (`left`, `right`, `up`, `down`) |
| `duration` | `number` | - | 애니메이션 지속 시간(ms) |

---

## 3. 혼합 연출 (Mixed Transitions)

`screen-fade`와 `screen-wipe`는 내부적으로 동일한 화면 전환 객체를 공유합니다. 따라서 한 효과로 화면을 덮고, 다른 효과로 화면을 걷어내는 등의 **교차 연출**이 가능합니다.

### 연속 연출 예시
```ts
[
  // 1. 검은색으로 부드럽게 페이드아웃 (화면 가림)
  { type: 'screen-fade', dir: 'out', preset: 'black', duration: 1000 },
  
  { type: 'dialogue', text: '잠시 후...' },

  // 2. 오른쪽에서 왼쪽 방향으로 와이프하며 화면 나타남 (화면 드러냄)
  // 페이드에서 설정한 'black' 색상이 와이프 효과에도 동일하게 적용됩니다.
  { type: 'screen-wipe', dir: 'in', preset: 'left', duration: 800 }
]
```

### 💡 주의: 플래시(Flash)는 예외
`screen-flash`는 화면 전환을 목적으로 하지 않는 **일시적인 특수 효과**입니다. 페이드나 와이프로 화면이 덮여 있는 상태와 관계없이 독립적으로 작동하며, 전환 상태를 변경하지 않습니다.

---

## 5. 레이어 및 우선순위 (Z-Index)

`screen` 모듈의 전환 효과(Fade, Wipe)는 내부적으로 **`zIndex: 300`**에서 작동합니다. 이는 배경과 캐릭터 등 연출 요소는 가리되, 플레이어에게 전달되는 중요 텍스트 정보(대화창, 오버레이)는 차단하지 않도록 설계되어 있습니다.

| 대상 레이어 | zIndex | 덮기 여부 | 비고 |
| :--- | :---: | :---: | :--- |
| **시스템 다이얼로그** | `700` | ❌ 미차단 | 선택지, 입력창, 알림창 등 |
| **캐릭터 컷인** | `600` | ❌ 미차단 | 특수 연출용 컷인 |
| **오버레이 (Overlay)** | `500` | ❌ 미차단 | 타이틀, 캡션 등 |
| **대화창 (Dialogue)** | `400` | ❌ 미차단 | 대화창 및 화자 이름 |
| **화면 전환 (Screen)** | **`300`** | - | **현재 레이어** |
| **환경 효과 (Env)** | `200` | ✅ 차단 | 조명, 무드 효과 등 |
| **캐릭터 (Char)** | `100` | ✅ 차단 | 일반 캐릭터 일러스트 |
| **배경 (Bg)** | `0` | ✅ 차단 | 배경 이미지/비디오 |

### 연출 팁
* **텍스트 유지**: 화면을 어둡게 페이드아웃(`screen-fade`)하더라도 현재 진행 중인 대사와 대화창은 그대로 유지됩니다. 이는 화면 전환 중에 대사를 읽게 하거나 전환의 흐름 속에서도 서사를 이어갈 때 유용합니다.
* **대화창 가리기**: 만약 대화창까지 가리고 싶다면, 대화창 모듈의 `action: 'hide'` 등을 함께 사용하여 연출하십시오.
* **플래시 우선순위**: `screen-flash`는 `301`로 작동하여 페이드나 와이프보다 아주 근소하게 상단에 위치합니다.

---

## 6. 주의 사항 (Edge Cases)

*   **입력 차단**: `disable: true` 설정 시 페이드나 와이프가 진행되는 동안 플레이어의 클릭 입력을 무시하여 연출의 몰입감을 보호할 수 있습니다.
*   **상태 보존**: `preset`을 생략하거나 `'inherit'`을 사용하면, 마지막으로 `screen-fade`나 `screen-wipe`에서 사용했던 색상이 그대로 유지됩니다.
