# 🎬 Screen Module

본 문서는 `screen` 모듈을 사용하여 화면 전체를 대상으로 하는 시각적 전환 효과 및 특수 연출을 제어하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`screen` 모듈은 장면과 장면 사이의 연결을 부드럽게 하거나 극적인 긴장감을 부여하는 화면 전체 효과를 담당합니다.  
화면을 특정 색상으로 덮거나 드러내는 **페이드(Fade)**, 순간적인 광휘를 발산하는 **플래시(Flash)**, 그리고 방향성 있는 전환을 구현하는 **와이프(Wipe)** 기능을 제공합니다.  

---

## 2. 주요 커맨드 상세 가이드

### 2.1. 화면 페이드 (`screen-fade`)
화면 전체를 특정 색상으로 서서히 가리거나(`out`), 가려진 상태에서 다시 원래의 화면을 드러내는(`in`) 효과입니다.  

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `dir` | `'in' \| 'out'` | ✅ | 페이드의 방향을 지정합니다.  (`in`: 나타남, `out`: 사라짐) |
| `preset` | `string` | - | 색상 프리셋을 선택합니다.  (`black`, `white`, `red`, `dream`, `sepia`) |
| `duration` | `number` | - | 애니메이션이 진행될 지속 시간(ms)입니다. |
| `disable` | `boolean` | - | 효과가 진행되는 동안 사용자의 클릭 입력을 차단할지 여부입니다. |

### 2.2. 화면 플래시 (`screen-flash`)
화면을 순간적으로 번쩍이게 하여 카메라 셔터 효과나 강한 타격감을 연출할 때 사용합니다.  

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `preset` | `string` | - | 플래시의 색상 프리셋을 선택합니다.  (`white`, `red`, `yellow`) |
| `duration` | `number` | - | 한 번의 플래시가 번쩍이는 지속 시간(ms)입니다. |
| `repeat` | `number` | - | 효과의 반복 횟수를 지정합니다. |

### 2.3. 화면 와이프 (`screen-wipe`)
그라데이션 마스크를 사용하여 화면을 특정 방향으로 닦아내듯 전환하는 세련된 효과를 제공합니다.  

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `dir` | `'in' \| 'out'` | ✅ | 와이프의 방향을 지정합니다.  (`in`: 나타남, `out`: 사라짐) |
| `preset` | `string` | - | 진행 방향 프리셋을 선택합니다.  (`left`, `right`, `up`, `down`) |
| `duration` | `number` | - | 애니메이션이 진행될 지속 시간(ms)입니다. |

---

## 3. 혼합 및 교차 연출 (Transition Blending)

`screen-fade`와 `screen-wipe`는 엔진 내부에서 동일한 전환 객체를 공유하도록 설계되어 있습니다.  
따라서 하나의 효과로 화면을 덮고 다른 방식의 효과로 화면을 걷어내는 **교차 연출**이 가능합니다.  

### 연속 연출 예시
```ts
[
  // 1. 검은색으로 부드럽게 페이드아웃하여 화면을 가립니다.  
  { type: 'screen-fade', dir: 'out', preset: 'black', duration: 1000 },
  
  { type: 'dialogue', text: '잠시 후...' },

  // 2. 오른쪽에서 왼쪽 방향으로 와이프하며 장면을 다시 드러냅니다.  
  // 페이드 시 설정했던 'black' 색상이 와이프 연출에도 자연스럽게 계승됩니다.  
  { type: 'screen-wipe', dir: 'in', preset: 'left', duration: 800 }
]
```

## 4. 레이어 구조 및 렌더링 우선순위 (Z-Index)

`screen` 모듈의 전환 효과는 엔진 내부의 **`zIndex: 300`** 레이어에서 작동합니다.  
이는 캐릭터나 배경과 같은 연출 요소는 가리되, 사용자에게 전달되어야 하는 중요 정보(대화창, 시스템 UI)는 시각적으로 차단하지 않도록 세밀하게 설계된 결과입니다.  

| 대상 레이어 그룹 | zIndex | 가림 여부 | 비고 |
| :--- | :---: | :---: | :--- |
| **시스템 다이얼로그** | `700` | ❌ 노출 유지 | 선택지, 입력창, 알림창 등 |
| **대화창 (Dialogue)** | `400` | ❌ 노출 유지 | 대화창 배경 및 텍스트 |
| **화면 전환 (Screen)** | **`300`** | - | **현재 모듈 레이어** |
| **환경 및 캐릭터** | `200~0` | ✅ 시각적 차단 | 조명, 캐릭터 레이어, 배경 이미지 |

---

## 5. 주의 사항 (Edge Cases)

*   **몰입도 보호**: `disable: true` 옵션을 활용하면 화면 전환이 진행되는 결정적인 순간에 사용자의 예기치 않은 입력을 차단하여 연출의 완성도를 높일 수 있습니다.  
*   **색상 계승 정책**: `preset`을 생략할 경우, 가장 마지막에 수행되었던 화면 전환 효과의 색상 정보를 그대로 유지하여 시각적 연속성을 보장합니다.  
*   **플래시의 독립성**: `screen-flash`는 화면 전환 상태와 무관하게 독립적으로 작동하며, 전환 레이어보다 근소하게 상위(`zIndex: 301`)에서 실행되어 가시성을 확보합니다.
