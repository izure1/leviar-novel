# 📍 Label Module

---

## 1. 개요 (Overview)

`label` 모듈은 시나리오 스크립트 내에서 특정 실행 지점을 가리키는 **마커(Marker)** 역할을 합니다. 이 커맨드 자체는 화면에 아무것도 출력하지 않으며 엔진 동작에도 즉각적인 영향을 주지 않지만, `condition` 모듈의 `goto` 속성이나 엔진의 점프 기능을 통해 시나리오 흐름을 제어할 때 중요한 참조점이 됩니다.

### 주요 특징
*   **실행 흐름 제어**: 특정 구간을 반복하거나, 선택지에 따라 일부 연출을 건너뛰고 특정 지점부터 시작하게 할 수 있습니다.
*   **비가시적 커맨드**: 사용자에게는 보이지 않으며, 스크립트 실행 중 라벨을 만나면 즉시 다음 커맨드로 넘어갑니다.

---

## 2. 핵심 예제 (Main Example)

### 반복 구간 설정 (Loop)

```ts
[
  // 1. 반복의 시작점을 라벨로 지정
  { type: 'label', name: 'practice_loop' },

  { type: 'dialogue', text: '칼 휘두르기 연습을 합니다. (슈슉-)' },

  // 2. 조건에 따라 라벨로 되돌아감
  { 
    type: 'condition', 
    if: ({ vars }) => vars.stamina > 0, 
    goto: 'practice_loop' 
  },

  { type: 'dialogue', text: '지쳐서 더 이상 연습할 수 없습니다.' }
]
```

---

## 3. 커맨드 상세 (Command Reference)

### Label 명령 (`LabelCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'label'` | ✅ | 라벨 커맨드임을 명시합니다. |
| `name` | `string` | ✅ | 해당 지점을 식별하기 위한 고유 이름입니다. `goto` 명령에서 이 이름을 사용해 이동합니다. |

---

## 4. 주의 사항 (Edge Cases)

*   **중복 이름**: 하나의 씬(Scene) 내에서 동일한 이름의 라벨을 여러 개 정의하면 안 됩니다. 엔진은 가장 먼저 찾은 라벨로 이동하므로 의도치 않은 동작이 발생할 수 있습니다.
*   **씬 간 이동 불가**: `label`은 현재 진행 중인 씬 내에서의 이동만을 지원합니다. 다른 씬으로 이동하고 싶다면 `condition` 모듈의 `next` 속성을 사용해야 합니다.
*   **배치 순서**: `goto` 명령이 실행되기 전에 라벨이 정의되어 있어야 하는 것은 아닙니다. 스크립트 상에서 `goto`보다 뒤에 있는 라벨로도 점프(Skip)가 가능합니다.
