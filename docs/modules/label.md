# 📍 Label Module

본 문서는 `label` 모듈을 사용하여 시나리오 스크립트 내에 참조 지점을 생성하고 시퀀스의 흐름을 제어하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`label` 모듈은 시나리오 스크립트 내에서 특정 실행 위치를 가리키는 **마커(Marker)** 역할을 수행합니다.  
이 커맨드는 시각적인 요소를 출력하지 않으며 엔진 동작에도 즉각적인 영향을 미치지 않으나, `condition` 모듈의 점프 기능이나 엔진의 시나리오 제어 시스템이 특정 지점을 식별하기 위한 필수적인 이정표가 됩니다.  

### 주요 특징
*   **지능형 흐름 제어**: 특정 연출 구간을 반복하거나, 플레이어의 선택에 따라 시나리오의 일부를 건너뛰고 목표 지점부터 실행을 재개할 수 있습니다.  
*   **비가시적 실행**: 사용자에게는 어떠한 시각적 피드백도 제공하지 않으며, 스크립트 실행 중 라벨을 조우하면 즉시 다음 커맨드로 순차 진행됩니다.  

---

## 2. 핵심 예제 (Main Example)

### 반복 시퀀스 설정 (Loop Implementation)
라벨과 조건문을 결합하여 특정 연출이 조건 만족 시까지 반복되도록 구성하는 예시입니다.  

```ts
[
  // 1. 반복이 시작될 지점을 라벨로 선언합니다.  
  { type: 'label', name: 'practice_loop' },

  { type: 'dialogue', text: '무술 수련을 지속합니다.  (검을 휘두르는 소리...)' },

  // 2. 현재 상태를 판별하여 지정된 라벨 지점으로 되돌아갑니다.  
  { 
    type: 'condition', 
    if: ({ vars }) => vars.stamina > 0, 
    goto: 'practice_loop' 
  },

  { type: 'dialogue', text: '체력이 모두 소진되어 수련을 종료합니다.' }
]
```

---

## 3. 커맨드 상세 (Command Reference)

### Label 명령 (`LabelCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'label'` | ✅ | 라벨 커맨드임을 명시합니다. |
| `name` | `string` | ✅ | 해당 위치를 식별하기 위한 고유 명칭입니다.  점프 명령에서 이 이름을 참조하여 이동합니다. |

---

## 4. 주의 사항 (Edge Cases)

*   **식별자 고유성 유지**: 하나의 씬(Scene) 파일 내에서 동일한 이름의 라벨을 중복하여 정의하는 것을 지양해 주십시오.  엔진은 검색 시 가장 먼저 발견된 라벨을 목표로 삼으므로, 의도치 않은 지점으로 이동할 위험이 있습니다.  
*   **스코프 제한**: `label`을 통한 이동은 현재 실행 중인 씬 내부의 범위를 벗어날 수 없습니다.  다른 씬 파일로의 전환을 원하실 경우에는 `condition` 모듈의 `next` 속성 또는 엔진의 씬 로드 기능을 활용하시기 바랍니다.  
*   **선언적 유연성**: 라벨의 위치는 이동 명령(`goto`)보다 앞서거나 뒤에 있어도 무방합니다.  엔진은 씬 전체의 인덱스를 분석하여 순방향 점프와 역방향 루프를 모두 안정적으로 지원합니다.
