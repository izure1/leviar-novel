# 🗳️ Choices Module

본 문서는 `choices` 모듈을 사용하여 사용자에게 선택지를 제시하고 시나리오의 흐름을 분기시키는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`choices` 모듈은 비주얼 노벨의 가장 중요한 특징인 인터랙티브한 분기 시스템을 담당합니다.  
사용자의 선택에 따라 시나리오를 다른 장면으로 연결하거나, 변수를 조작하여 이후의 전개에 영향을 줄 수 있는 정교한 기능을 제공합니다.  

### 주요 특징
*   **유연한 시나리오 분기**: 선택 결과에 따라 새로운 씬(`next`)으로 이동하거나 현재 씬 내의 특정 지점(`goto`)으로 즉시 점프할 수 있습니다.  
*   **지능형 변수 조작**: 선택 즉시 전역 또는 지역 변수의 값을 변경하여 게임의 상태를 실시간으로 업데이트합니다.  
*   **동적 텍스트 생성**: 함수 형태의 텍스트 정의를 지원하여 변수 값에 따라 선택지의 문구를 유동적으로 변화시킬 수 있습니다.  

---

## 2. 핵심 예제 (Main Example)

### 상호작용 및 분기 구현
변수 상태를 실시간으로 반영하며, 선택 결과에 따라 데이터 연산과 장면 전환을 동시에 수행하는 예시입니다.  

```ts
{
  type: 'choices',
  choices: [
    { 
      // 1. 동적 텍스트: {{ }} 보간법 또는 함수형 반환을 활용합니다.  
      text: '뇌물을 건넨다 (현재 소지금: {{gold}}G)', 
      // 2. 씬 전환: 지정된 새로운 씬 파일로 즉시 이동합니다.  
      next: 'bribe_success_scene',
      // 3. 동적 변수 조작: Resolvable 함수를 통해 현재 소지금을 차감합니다.  
      var: ({ gold }) => ({ gold: gold - 100 })
    },
    { 
      text: ({ heart }) => heart > 50 ? '정중하게 부탁한다 (호감도 높음)' : '그냥 지나간다',
      goto: 'next_label',
      var: { heart: 5 } // 단순 값 할당 방식입니다.  
    }
  ]
}
```

## 3. 커맨드 상세 (Command Reference)

### Choices 명령 (`ChoiceCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'choices'` | ✅ | 선택지 커맨드임을 명시합니다. |
| `choices` | `ChoiceItem[]` | ✅ | 제시할 선택지 객체들의 배열입니다. |

#### ChoiceItem 객체 상세

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `text` | `string \| Function` | ✅ | 버튼에 표시될 텍스트입니다. |
| `next` | `string \| Object` | - | 선택 시 이동할 다음 씬의 이름 또는 설정 객체입니다. |
| `goto` | `string` | - | 현재 씬 내에서 이동할 라벨(`label`)의 이름입니다. |
| `var` | `Object \| Function` | - | 선택 시 업데이트할 변수 설정입니다.  함수형 객체를 통한 동적 연산을 지원합니다. |

## 5. 주의 사항 (Edge Cases)

*   **사용자 입력 제어**: 선택지 커맨드가 실행되는 동안은 대사 진행을 위한 일반 클릭 입력이 자동으로 차단됩니다.  
*   **씬 전환의 즉시성**: `next` 속성이 지정된 경우 현재 실행 중인 씬은 즉시 소거되고 새로운 씬의 컨텍스트가 로드됩니다.  
*   **동적 문구 처리**: `text` 속성에 함수를 사용할 경우 실시간 데이터 상태를 기반으로 매번 새로운 문구를 생성하므로 연출의 유연성이 극대화됩니다.
