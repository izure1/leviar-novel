# 🎨 Input State & Layout

본 문서는 `input` 모듈의 시각적 구성 요소(배경 레이어, 입력 패널, 커서, 버튼)와 배치 레이아웃을 결정하는 `InputSchema`에 대해 기술합니다.  

---

## 1. 개요 (Overview)

입력 모듈의 상태 설정은 플레이어가 정보를 기입하는 인터페이스의 심미성과 사용성을 정의합니다.  
반투명한 배경 처리부터 깜빡이는 커서의 형태까지 정교하게 제어할 수 있으며, 이를 통해 게임의 UI 테마와 완벽하게 조화되는 입력 환경을 조성할 수 있습니다.  

| 속성 그룹 명칭 | 상세 설명 |
| :--- | :--- |
| **`overlay`** | 입력창 활성화 시 뒷배경 전체를 덮어 집중도를 높이는 반투명 레이어 스타일입니다. |
| **`panel`** | 입력 필드와 버튼이 담기는 메인 컨테이너 패널의 시각적 스타일입니다. |
| **`labelStyle`** | 입력창 상단에 표시되는 안내 문구(Label)의 서체 및 색상 설정입니다. |
| **`inputTextStyle`** | 플레이어가 실제로 기입하고 있는 텍스트의 스타일 설정입니다. |
| **`cursorStyle`** | 입력 위치를 알리는 깜빡이는 커서(`|`)의 시각적 형태입니다. |
| **`button`** | 하단에 배치되는 확인 및 취소 버튼들의 기본적인 배경 스타일입니다. |
| **`layout`** | 패널 내부의 정밀한 여백과 구성 요소 간의 간격을 정의합니다. |

---

## 2. 레이아웃 상세 명세 (`InputLayout`)

입력창 구성 요소 간의 물리적 배치를 조절하여 쾌적한 입력 환경을 제공합니다.  

| 속성 명칭 | 데이터 타입 | 기본값 | 설명 |
| :--- | :--- | :---: | :--- |
| **`panelPaddingLeft`** | `number` | `32` | 패널 내부 좌측 끝에서 콘텐츠 시작점까지의 여백(px)입니다. |
| **`panelPaddingRight`** | `number` | `32` | 패널 내부 우측 끝까지의 여백입니다. |
| **`panelPaddingTop`** | `number` | `24` | 패널 상단 경계에서 레이블 텍스트까지의 거리입니다. |
| **`panelPaddingBottom`** | `number` | `24` | 버튼 영역 하단에서 패널 경계까지의 여백입니다. |
| **`buttonPaddingLeft`** | `number` | `20` | 각 버튼 내부의 수평 여백입니다. |
| **`buttonPaddingRight`** | `number` | `20` | 각 버튼 내부의 수평 여백입니다. |
| **`buttonPaddingTop`** | `number` | `8` | 각 버튼 내부의 수직 여백입니다. |
| **`buttonPaddingBottom`** | `number` | `8` | 각 버튼 내부의 수직 여백입니다. |
| **`labelInputGap`** | `number` | `12` | 안내 문구와 입력 필드 사이의 수직 간격입니다. |
| **`inputButtonGap`** | `number` | `20` | 입력 필드 하단과 버튼 영역 사이의 충분한 분리 공간입니다. |
| **`buttonGap`** | `number` | `8` | 버튼과 버튼 사이의 가로 여백입니다. |

## 3. 스타일링 활용 사례

### 모던한 다크 모드 테마의 입력창 디자인
시인성이 뛰어난 어두운 톤의 입력 인터페이스를 구성하는 예시입니다.  

```ts
export default defineScene({
  initial: {
    input: {
      panel: {
        color: '#1a1a1a', // 묵직한 다크 그레이 배경
        borderColor: '#3498db',
        borderWidth: 2,
        borderRadius: 12,
        minWidth: 500 // 충분한 가로 너비 확보
      },
      inputTextStyle: {
        fontSize: 24,
        color: '#3498db',
        fontWeight: 'bold'
      },
      button: {
        color: '#3498db',
        borderRadius: 6
      }
    }
  }
})
```

## 4. 상세 속성 가이드

### 배경 오버레이 (`overlay`)
사용자가 입력을 시작할 때 주변 시각 요소를 차단하여 집중을 유도합니다.  
*   **`color`**: 배경색을 지정합니다.  (예: `rgba(0,0,0,0.6)`)  

### 가상 커서 설정 (`cursorStyle`)
입력 중임을 알리는 애니메이션 객체(`|`)의 스타일을 지정합니다.  
*   일반적으로 `inputTextStyle`의 색상 및 폰트 크기와 일치시켜 시각적 통일성을 확보합니다.  

### 버튼 스타일링 (`button` & `buttonHover`)
모든 버튼에 적용될 기본 외형을 정의합니다.  개별 버튼의 특수한 스타일은 커맨드 레벨의 `buttons` 속성에서 직접 오버라이드할 수 있습니다.  

---

## 5. 주의 사항 (Edge Cases)

*   **최소 너비 정책**: 입력 패널은 내부 텍스트 길이에 따라 높이가 가변적으로 변하지만, 너비는 기본적으로 `420px` 이상의 최소 너비를 엄격히 유지하여 레이아웃 붕괴를 방지합니다.  
*   **자동 중앙 정렬**: 입력 패널은 엔진에 의해 항상 화면의 기하학적 정중앙에 자동으로 배치되도록 설계되었습니다.  
*   **상태 영속성**: 런타임에 동적으로 변경된 모든 입력창 스타일은 엔진의 상태 시스템에 기록되어, 세이브 및 로드 시에도 동일한 시각적 경험을 제공합니다.
