# ⚙️ Choices State & Layout

본 문서는 `choices` 모듈의 시각적 형태(Style)와 배치(Layout)를 결정하는 데이터 구조인 `ChoiceSchema`에 대해 기술합니다.  

---

## 1. 개요 (Overview)

선택지 모듈의 상태 설정은 플레이어에게 제시되는 선택지 버튼의 미학적 요소와 정렬 방식을 정의합니다.  
`defineScene`의 `initial` 섹션을 통해 씬 단위의 기본 스타일을 지정하거나, 런타임에 `ui` 커맨드를 활용하여 동적으로 외형을 변경함으로써 상황에 맞는 최적의 사용자 경험을 제공할 수 있습니다.  

---

## 2. 상태 상세 명세 (`ChoiceSchema`)

| 속성 명칭 | 데이터 타입 | 상세 설명 |
| :--- | :--- | :--- |
| **`bg`** | `Partial<Style>` | 선택지 활성화 시 화면 전체를 덮는 배경(오버레이)의 스타일입니다. |
| **`button`** | `Partial<Style>` | 선택지 버튼의 기본적인 배경색, 테두리, 그림자 등의 스타일입니다. |
| **`buttonHover`** | `Partial<Style>` | 사용자의 마우스 커서가 버튼 위에 위치할 때 적용되는 강조 스타일입니다. |
| **`text`** | `Partial<Style>` | 버튼 내부에 표시되는 텍스트의 폰트, 크기, 색상 설정입니다. |
| **`textHover`** | `Partial<Style>` | 호버 상태에서 텍스트에 적용되는 시각적 변화 설정입니다. |
| **`layout`** | `ChoiceLayout` | 버튼 간의 간격, 패딩 등 기하학적인 배치 규칙을 정의합니다. |

---

## 3. 레이아웃 상세 설정 (`ChoiceLayout`)

버튼 객체들 사이의 거리와 내부 여백을 정밀하게 제어하여 균형 잡힌 레이아웃을 구성합니다.  

| 속성 명칭 | 데이터 타입 | 기본값 | 설명 |
| :--- | :--- | :---: | :--- |
| **`gap`** | `number` | `12` | 선택지 버튼들 사이의 세로 여백(px)을 결정합니다. |
| **`buttonPaddingLeft`** | `number` | `32` | 버튼 내부 좌측 끝에서 텍스트까지의 여백입니다. |
| **`buttonPaddingRight`** | `number` | `32` | 버튼 내부 우측 끝에서 텍스트까지의 여백입니다. |
| **`buttonPaddingTop`** | `number` | `12` | 버튼 상단 테두리에서 텍스트까지의 여백입니다. |
| **`buttonPaddingBottom`** | `number` | `12` | 버튼 하단 테두리에서 텍스트까지의 여백입니다. |

---

## 4. 활용 사례 (Examples)

### 씬 진입 시 초기 디자인 설정
장면의 성격에 맞춰 선택지의 디자인 테마를 사전에 정의하는 방식입니다.  

```ts
export default defineScene({
  initial: {
    choices: {
      button: { color: '#2c3e50', borderRadius: 20 },
      text: { fontSize: 22, color: '#ecf0f1' },
      layout: { gap: 20 }
    }
  }
})
```

---

## 5. 주의 사항 (Edge Cases)

*   **지능형 상태 병합**: `ui` 커맨드를 통해 런타임에 상태를 변경할 경우, 명시적으로 지정한 속성값만 업데이트되며 나머지 기존 설정값들은 안전하게 보존됩니다.  
*   **레이아웃 결정 우선순위**: `button` 스타일 내에서 `width`를 직접 고정값으로 지정하면, 패딩 기반의 자동 너비 계산 로직보다 우선하여 적용됩니다.  
*   **호버 상태의 일관성**: 사용자 인터랙션의 명확성을 위해 가급적 `buttonHover` 스타일을 기본 `button` 스타일과 대비되도록 설정해 주시기를 권장합니다.  
*   **영속성 보존**: 현재 적용된 모든 스타일 및 레이아웃 상태는 엔진의 직렬화 시스템에 포함되어 세이브 데이터에 반영됩니다.
