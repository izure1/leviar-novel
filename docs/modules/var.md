# 🔢 Variable Module (`var`)

본 문서는 `var` 모듈을 사용하여 게임 내 변수의 상태를 동적으로 제어하고 관리하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`var` 모듈은 게임의 진행 상태를 결정짓는 데이터를 조작하는 핵심 기능을 담당합니다.  
`fumika` 엔진은 데이터의 유효 범위를 기준으로 **전역 변수(Global)**와 **지역 변수(Local)**라는 두 가지 스코프를 지원하며, 각 변수의 생명주기는 다음과 같이 관리됩니다.  

*   **전역 변수 (Global)**: 모든 씬에서 공유되는 영구적인 데이터입니다.  
*   **지역 변수 (Local, `_` 접두사)**: 현재 실행 중인 씬 내에서만 유효하며, 씬이 전환되거나 종료될 때 자동으로 초기화됩니다.  

엔진은 플레이어의 진행 상황을 완벽하게 보존하기 위해, **전역 변수와 현재 활성화된 씬의 지역 변수를 모두 세이브 파일에 포함하여 관리합니다.**  

---

## 2. 커맨드 상세 (Command Reference)

### Variable 설정 명령 (`VarCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'var'` | ✅ | 커맨드 타입을 명시합니다. |
| `name` | `string` | ✅ | 변경하고자 하는 변수의 식별자입니다.  `_`로 시작할 경우 지역 변수로 처리됩니다. |
| `value` | `any \| Function` | ✅ | 설정할 새로운 값입니다.  단순 값 또는 동적 연산을 위한 함수를 수용합니다. |

---

## 3. 활용 예시 (Examples)

### 3.1. 전역 상태 관리
플레이어의 성향이나 게임 진행도를 영구적으로 기록할 때 사용합니다.  

```ts
[
  { type: 'var', name: 'affection', value: 50 },
  { type: 'var', name: 'has_item_key', value: true },
  { type: 'dialogue', text: '대상이 당신에게 호감을 느끼기 시작했습니다.' }
]
```

### 3.2. 씬 전용 임시 상태 관리
특정 장면 내에서의 루프 카운트나 일회성 플래그를 관리할 때 `_` 접두사를 활용해 주십시오.  

```ts
[
  // 해당 씬이 종료되면 자동으로 소거되는 지역 변수입니다.  
  { type: 'var', name: '_investigation_count', value: 1 },
  { type: 'dialogue', text: '주변을 세밀하게 조사하였습니다.' }
]
```

## 4. 고급 활용: Resolvable 연산 (Dynamic Updates)

`value` 속성에 단순한 값이 아닌 **함수(Resolvable)**를 전달함으로써, 현재 저장된 데이터 상태를 기반으로 한 지능적인 연산을 수행할 수 있습니다.  

### 4.1. 수치 가감 및 범위 제한 (Clamping)
현재 값을 참조하여 수치를 증가시키거나, 특정 범위를 벗어나지 않도록 제어할 수 있습니다.  

```ts
{
  type: 'var',
  name: 'hp',
  // 현재 HP 상태를 참조하여 최대치(100)를 초과하지 않도록 회복합니다.  
  value: ({ hp }) => Math.min(100, hp + 20)
}
```

### 4.2. 논리 반전 및 복합 참조
변수의 불리언 상태를 토글하거나 여러 변수의 값을 조합하여 새로운 데이터를 산출할 수 있습니다.  

```ts
{
  type: 'var',
  name: 'luck_factor',
  // 지력(int)과 행운(luk)의 평균값에 지역 보너스를 더해 동적으로 계산합니다.  
  value: ({ int, luk, _bonus }) => ((int + luk) / 2) + _bonus
}
```

---

## 5. 주의 사항 (Edge Cases)

*   **데이터 타입 준수**: `value`는 JavaScript의 모든 기본 타입을 수용하나, 가급적 `novel.config.ts`에서 정의한 초기 타입 구조를 유지하는 것이 유지보수에 유리합니다.  
*   **세이브 데이터 포함 여부**: 모든 변수명과 값은 `novel.save()` 호출 시 엔진에 의해 자동으로 직렬화되어 보존됩니다.  
*   **스코프 격리**: 지역 변수는 씬 전환 시 메모리 효율을 위해 자동으로 소거되므로, 씬 간 데이터 공유가 필요한 경우에는 반드시 전역 변수명을 사용해 주시기 바랍니다.
