# 🎭 Mood Module

본 문서는 `mood` 모듈을 사용하여 화면 전체의 조명, 색감 및 분위기를 효과적으로 연출하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`mood` 모듈은 화면 전반에 컬러 필터, 비네팅(Vignette), 동적 조명 효과 등을 적용하여 게임의 시간적 배경과 감정적 분위기를 시각화합니다.  
단순한 색상 중첩을 넘어 다양한 블렌드 모드(Multiply, Screen 등)와 실시간 조명 깜빡임(Flicker) 연출을 결합하여, 플레이어에게 깊은 몰입감을 선사합니다.  

### 주요 특징
*   **풍부한 환경 프리셋**: 낮, 밤, 노을, 회상, 공포 등 15종 이상의 검증된 분위기 설정을 즉시 적용할 수 있습니다.  
*   **지능형 블렌드 엔진**: 각 무드 프리셋에 최적화된 합성 모드가 자동으로 활성화되어, 배경 및 캐릭터와 이질감 없이 자연스럽게 어우러집니다.  
*   **역동적인 조명 애니메이션**: 촛불의 일렁임이나 노후된 전등의 깜빡임 등 생동감 있는 물리 효과를 지원합니다.  
*   **다중 레이어 중첩**: 여러 개의 무드를 레이어 방식으로 쌓아 올려, 더욱 복잡하고 세밀한 시각적 분위기를 창조할 수 있습니다.  

---

## 2. 핵심 예제 (Main Example)

### 시간대 전환 및 분위기 고조 연출 구현

```ts
[
  // 1. 기본적인 환경 무드 설정 (노을 분위기 적용)
  { 
    type: 'mood', 
    mood: 'sunset', 
    intensity: 0.4, 
    duration: 2000 
  },

  { type: 'dialogue', text: '창밖으로 붉은 노을이 길게 드리우기 시작했습니다.' },

  // 2. 기존 무드 위에 특수 효과 중첩 (공포 분위기 가미)
  { 
    type: 'mood', 
    action: 'add', 
    mood: 'horror', 
    intensity: 0.6, 
    flicker: 'flicker' // 조명이 불안정하게 깜빡이는 효과를 추가합니다.  
  },

  { type: 'dialogue', text: '갑자기 실내의 공기가 차갑게 식으며, 머리 위의 전등이 불안하게 흔들립니다.' },

  // 3. 모든 환경 효과 제거 및 기본 상태 복구
  { type: 'mood', mood: 'none', duration: 1000 }
]
```

## 3. 커맨드 상세 (Command Reference)

### Mood 명령 (`MoodCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'mood'` | ✅ | 무드 커맨드임을 명시합니다. |
| `action` | `'add' \| 'remove'` | - | 수행할 동작 유형입니다.  생략 시 기존 무드를 모두 제거하고 새로운 무드만 적용합니다. |
| `mood` | `string` | ✅ | 적용하고자 하는 무드의 프리셋 키입니다. |
| `intensity` | `number` | - | 효과의 투명도 및 강도(0.0 ~ 1.0)를 지정합니다. |
| `flicker` | `string` | - | 조명의 일렁임이나 깜빡임 프리셋을 적용합니다.  (`candle`, `flicker`, `strobe`) |
| `duration` | `number` | - | 분위기 전환이 완료될 때까지의 시간(ms)입니다. |
| `disable` | `boolean` | - | `true` 설정 시 전환 애니메이션 도중 사용자의 입력을 일시 차단합니다. |

## 4. 주요 무드 프리셋 안내 (Mood Types)

| 프리셋 키 | 연출 분위기 | 권장 활용 상황 |
| :--- | :--- | :--- |
| `day` | 밝은 낮 | 평상시, 화창한 야외 |
| `night` | 어두운 밤 | 밤 시간대, 어두운 실내 |
| `dawn` | 새벽 | 해 뜨기 전, 푸르스름한 분위기 |
| `sunset` | 노을 | 해 질 녘, 주황빛 분위기 |
| `foggy` | 안개 | 흐린 날씨, 신비로운 장소 |
| `sepia` | 세피아 | 과거 회상, 오래된 사진 느낌 |
| `cold` | 차가움 | 냉정한 분위기, 겨울, 긴장감 |
| `noir` | 느와르 | 흑백 영화 느낌, 고독한 연출 |
| `horror` | 공포 | 붉고 어두운 분위기, 위기 상황 |
| `flashback` | 회상 | 밝고 뽀얀 느낌의 회상 장면 |
| `dream` | 꿈 | 몽환적이고 보랏빛이 도는 연출 |
| `danger` | 위험 | 강한 붉은색, 경고 상황 |
| `spot` | 스포트라이트 | 중앙 집중 조명 |
| `warm` | 따뜻함 | 포근한 실내, 행복한 분위기 |

## 5. 주의 사항 (Edge Cases)

*   **동작 모드 선택**: `action: 'add'`를 명시적으로 지정하지 않으면, 새로운 무드 적용 시 기존의 모든 무드 레이어가 자동으로 소거됩니다.  단일 분위기를 전환할 때는 속성을 생략하여 간결하게 사용하시기 바랍니다.  
*   **환경 초기화**: 모든 시각 효과를 제거하고 표준 렌더링 상태로 복귀하려면 `mood: 'none'`을 호출해 주십시오.  
*   **자원 관리**: 지나치게 많은 무드 레이어를 중첩하여 사용하면 렌더링 부하가 증가할 수 있으므로, 시나리오의 의도에 맞게 적절한 수의 레이어를 유지하는 것을 권장합니다.  
*   **영속성 보장**: 현재 적용된 모든 무드 레이어 정보는 엔진의 상태 저장소에 기록되어 세이브 데이터로 안전하게 관리됩니다.
