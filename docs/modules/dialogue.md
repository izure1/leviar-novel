# 💬 Dialogue Module

본 문서는 `dialogue` 모듈을 사용하여 비주얼 노벨의 대사와 나레이션을 효과적으로 출력하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`dialogue` 모듈은 비주얼 노벨의 핵심 요소인 **대화 및 나레이션 출력**을 전담합니다.  
캐릭터의 성명과 대사를 화면 하단의 대화창에 표시하며, 글자가 하나씩 나타나는 타이핑 효과와 다양한 스타일 커스터마이징을 지원합니다.  

### 주요 특징
*   **타이핑 효과**: 텍스트가 한 글자씩 출력되는 연출을 제공하며, 출력 속도를 정교하게 조절할 수 있습니다.  
*   **다중 라인 처리**: 하나의 커맨드 안에 여러 줄의 대사를 배열 형태로 정의하여, 사용자 클릭에 따라 순차적으로 출력할 수 있습니다.  
*   **지능형 화자 해석**: `novel.config.ts`의 `characters`에 등록된 식별자를 `speaker`로 지정하면, 해당 캐릭터 정의에 설정된 실제 성함이 화면에 자동으로 출력됩니다.  
*   **실시간 변수 보간**: `{{var}}` 문법을 사용하여 게임 내 변수 값을 대사 중에 실시간으로 반영할 수 있습니다.  

---

## 2. 핵심 예제 (Main Example)

### 기본 대화 및 나레이션 구현

```ts
[
  // 1. 화자가 명시된 대화
  { type: 'dialogue', speaker: 'heroine', text: '안녕! 오늘 날씨가 정말 좋다.' },

  // 2. 나레이션 (speaker 속성 생략)
  { type: 'dialogue', text: '그녀는 밝게 웃으며 나에게 인사를 건넸다.' },

  // 3. 연속적인 대사 출력 (사용자 클릭 시마다 다음 줄 진행)
  { 
    type: 'dialogue', 
    speaker: 'heroine', 
    text: [
      '있잖아, 사실 할 말이 있어...',
      '나랑 같이 축제에 가지 않을래?'
    ],
    speed: 50 // 타이핑 속도 지정 (ms 단위)
  }
]
```

---

## 3. 커맨드 상세 (Command Reference)

### Dialogue 명령 (`DialogueCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'dialogue'` | ✅ | 대화 커맨드임을 명시합니다. |
| `text` | `string \| string[]` | ✅ | 출력할 대사 내용입니다.  배열일 경우 순차적으로 출력됩니다. |
| `speaker` | `string` | - | 화자의 식별자입니다.  등록된 캐릭터 키를 사용하면 실제 성함이 표시됩니다. |
| `speed` | `number` | - | 타이핑 속도(ms)입니다.  값이 작을수록 빠르며, 0일 경우 즉시 출력됩니다. |

---

## 5. 주의 사항 (Edge Cases)

*   **사용자 입력 반응**: 대사가 출력되는 도중 클릭이 발생하면 타이핑이 즉시 완료됩니다.  이후 다시 클릭해야 다음 단계로 진행됩니다.  
*   **나레이션 모드**: `speaker` 속성을 지정하지 않으면 이름 표시창이 자동으로 비활성화되며 나레이션 형태로 작동합니다.  
*   **보간 처리 순서**: 텍스트 내부의 `{{ }}` 보간 로직은 렌더링 훅이 실행되기 전에 최우선적으로 처리됩니다.
