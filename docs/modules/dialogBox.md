# 📦 DialogBox Module

---

## 1. 개요 (Overview)

`dialogBox` 모듈은 게임 내에서 플레이어에게 정보를 전달하거나, 확인을 받거나, 선택을 요구할 때 사용하는 **범용 메시지 박스**입니다. `choices` 모듈보다 좀 더 시스템적인 안내(알림, 경고, 확인창 등)에 적합한 디자인과 기능을 제공합니다.

### 주요 특징
*   **시스템 알림**: 제목과 본문을 가진 전형적인 윈도우 스타일의 대화상자를 제공합니다.
*   **유연한 버튼 배치**: 버튼의 개수에 따라 자동으로 행과 열을 계산하여 최적의 위치에 배치합니다.
*   **강제 선택 제어**: `persist` 옵션을 통해 플레이어가 반드시 버튼을 눌러야만 창을 닫을 수 있도록 강제할 수 있습니다.
*   **변수 조작 연동**: 버튼 클릭 시 특정 변수 값을 변경하는 로직을 직접 포함할 수 있습니다.

---

## 2. 핵심 예제 (Main Example)

### 알림 및 확인창 구성

```ts
[
  // 1. 단순 알림창 (확인 버튼 하나)
  { 
    type: 'dialogBox', 
    title: '시스템 알림', 
    content: '게임 데이터가 성공적으로 저장되었습니다.',
    buttons: [{ text: '확인' }]
  },

  // 2. 변수 조작이 포함된 확인창 (강제 선택)
  {
    type: 'dialogBox',
    title: '아이템 구매',
    content: '포션을 구매하시겠습니까? (100G 소모)',
    persist: true, // 창 바깥을 클릭해도 닫히지 않음
    buttons: [
      { 
        text: '구매하기', 
        var: ({ gold }) => ({ gold: gold - 100, potions: potions + 1 })
      },
      { text: '그만두기' }
    ]
  }
]
```

## 3. 커맨드 상세 (Command Reference)

### DialogBox 명령 (`DialogBoxCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'dialogBox'` | ✅ | 대화상자 커맨드임을 명시합니다. |
| `title` | `string` | ✅ | 상단에 표시될 제목입니다. |
| `content` | `string` | ✅ | 중앙에 표시될 본문 내용입니다. |
| `buttons` | `object[]` | ✅ | 하단 버튼 목록입니다. (상세 구성은 아래 참조) |
| `persist` | `boolean` | - | `true`이면 배경 클릭으로 창을 닫을 수 없습니다. **(버튼이 1개라도 있으면 자동으로 `true`가 됩니다.)** |
| `duration` | `number` | - | 창이 나타나고 사라지는 애니메이션 시간(ms)입니다. |

#### Buttons 객체 구성

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `text` | `string` | ✅ | 버튼에 표시될 텍스트입니다. |
| `var` | `Resolvable` | - | 클릭 시 실행될 변수 조작 로직입니다. |

## 6. 주의 사항 (Edge Cases)

*   **강제 선택(Forced Persist)**: 버튼이 하나라도 정의되어 있다면, `persist: false`로 설정하더라도 엔진 내부에서 자동으로 `true`로 처리됩니다. 이는 사용자가 실수로 배경을 클릭하여 선택 기회를 날리는 것을 방지하기 위함입니다.
*   **배경 클릭(Dismiss)**: 버튼이 전혀 없는 특수한 경우에만 `persist: false`가 유효하며, 이때 상자 바깥쪽(오버레이)을 클릭하면 인덱스 `-1`로 처리되며 창이 닫힙니다.
*   **버튼 자동 줄바꿈**: 버튼이 너무 많아 패널 너비를 초과하면 자동으로 다음 줄로 넘어가며 패널 높이가 늘어납니다.
*   **우선순위**: 대화상자가 활성화되어 있는 동안에는 일반적인 대사 진행(`dialogue`)이 차단됩니다.
