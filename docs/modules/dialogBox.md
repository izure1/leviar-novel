# 📦 DialogBox Module

본 문서는 `dialogBox` 모듈을 사용하여 플레이어에게 시스템 정보를 전달하고 선택을 요구하는 범용 메시지 박스를 구성하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`dialogBox` 모듈은 게임 내에서 플레이어에게 공식적인 정보를 안내하거나, 확인 및 경고와 같은 시스템적인 상호작용을 수행하는 창을 생성합니다.  
서사적인 분기를 담당하는 `choices` 모듈과 달리, 전형적인 윈도우 스타일의 제목과 본문 구조를 갖추고 있어 알림창이나 설정 확인창 등에 최적화된 기능을 제공합니다.  

### 주요 특징
*   **시스템 알림 구조**: 명확한 제목과 본문 텍스트를 통해 정보의 가시성을 극대화합니다.  
*   **지능형 버튼 레이아웃**: 배치된 버튼의 개수에 따라 자동으로 최적의 행과 열을 계산하여 균형 잡힌 화면을 구성합니다.  
*   **강제 선택 시스템**: `persist` 옵션을 통해 사용자가 반드시 지정된 버튼 중 하나를 클릭해야만 창을 닫을 수 있도록 제어할 수 있습니다.  
*   **변수 조작 연동**: 각 버튼 클릭 시 엔진 내부의 변수 값을 즉시 변경하는 로직을 통합하여 관리할 수 있습니다.  

---

## 2. 핵심 예제 (Main Example)

### 알림 및 구매 확인창 구현
플레이어에게 진행 상태를 알리거나, 변수 연산을 수반하는 선택을 유도하는 예시입니다.  

```ts
[
  // 1. 단순 시스템 알림 (확인 절차를 거치는 기본 형태입니다)
  { 
    type: 'dialogBox', 
    title: '데이터 보존 완료', 
    content: '게임 데이터가 시스템 저장소에 안전하게 기록되었습니다.',
    buttons: [{ text: '확인' }]
  },

  // 2. 변수 조작이 포함된 구매 확인창 (강제 선택 모드)
  {
    type: 'dialogBox',
    title: '아이템 획득',
    content: '회복 포션을 구매하시겠습니까? (100G가 소요됩니다)',
    persist: true, // 버튼 클릭 외의 외부 클릭으로 창이 닫히는 것을 방지합니다.  
    buttons: [
      { 
        text: '구매하기', 
        var: ({ gold, potions }) => ({ gold: gold - 100, potions: potions + 1 })
      },
      { text: '취소' }
    ]
  }
]
```

## 3. 커맨드 상세 (Command Reference)

### DialogBox 명령 (`DialogBoxCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'dialogBox'` | ✅ | 대화상자 커맨드임을 명시합니다. |
| `title` | `string` | ✅ | 상단 영역에 표시될 제목 문구입니다. |
| `content` | `string` | ✅ | 중앙 영역에 표시될 본문 상세 내용입니다. |
| `buttons` | `object[]` | ✅ | 하단에 배치할 버튼들의 구성 목록입니다. |
| `persist` | `boolean` | - | `true` 설정 시 배경 클릭을 통한 창 닫기를 제한합니다.  (버튼 존재 시 자동 활성화) |
| `duration` | `number` | - | 창이 나타나고 사라지는 애니메이션 지속 시간(ms)입니다. |

#### Buttons 객체 상세 구성

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `text` | `string` | ✅ | 버튼에 표시될 텍스트입니다. |
| `var` | `Resolvable` | - | 해당 버튼 클릭 시 실행될 엔진 변수 조작 로직입니다. |

## 5. 주의 사항 (Edge Cases)

*   **자동 강제 선택 정책**: 보안과 조작 무결성을 위해, 버튼이 하나라도 정의되어 있다면 엔진은 자동으로 `persist` 상태를 `true`로 격상시킵니다.  이는 사용자가 배경을 잘못 클릭하여 중요한 선택 기회를 유실하는 것을 방지하기 위함입니다.  
*   **동적 레이아웃 반응**: 버튼의 수량이 많아 패널의 너비를 초과할 경우, 엔진은 자동으로 행을 추가하여 패널의 높이를 확장합니다.  
*   **진행 흐름의 일시 차단**: 대화상자가 활성화되어 있는 동안에는 일반적인 대사 진행(`dialogue`)이 자동으로 차단되어, 사용자가 현재의 정보에 집중할 수 있도록 유도합니다.  
*   **배경 소거(Dismiss)의 유효성**: 버튼이 전혀 존재하지 않는 특수한 경우에만 `persist: false` 설정이 작동하며, 이때 상자 외부 영역을 클릭하면 창이 소거됩니다.
