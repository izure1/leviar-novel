# 🖥️ UI Control Module (`ui`)

본 문서는 `ui` 모듈을 사용하여 엔진에 등록된 다양한 UI 구성 요소의 가시성을 정교하게 제어하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`ui` 모듈은 대화창, 오버레이, 선택지 등 엔진의 화면을 구성하는 개별 UI 모듈들의 가시성 상태를 직접 조작하는 기능을 담당합니다.  
특정 연출 시점에 대화창을 잠시 소거하여 화면 전체의 몰입감을 높이거나, 숨겨져 있던 특수 UI를 시나리오 로직에 따라 강제로 노출해야 하는 상황에서 필수적으로 활용됩니다.  

---

## 2. 커맨드 상세 (Command Reference)

### UI 제어 명령 (`UICmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'ui'` | ✅ | 커맨드 타입을 명시합니다. |
| `name` | `string` | ✅ | 제어하고자 하는 대상 모듈의 식별자입니다.  (`config`에 등록된 키 이름) |
| `action` | `'show' \| 'hide'` | ✅ | 모듈을 화면에 표시할지(`show`), 숨길지(`hide`) 결정합니다. |
| `duration` | `number` | - | 나타나거나 사라질 때 적용되는 전환 애니메이션의 지속 시간(ms)입니다. |

---

## 3. 활용 예시 (Usage Examples)

### 3.1. 감성 연출을 위한 대화창 일시 소거
배경의 아름다움이나 캐릭터의 전신 연출을 강조하기 위해 대화창을 잠시 숨기는 일반적인 기법입니다.  

```ts
[
  { type: 'dialogue', text: '이곳의 정경은 정말이지 말로 표현할 수 없을 만큼 아름답군요.' },
  
  // 대화창을 0.5초간 서서히 숨깁니다.  
  { type: 'ui', name: 'dialogue', action: 'hide', duration: 500 },
  
  { type: 'wait', duration: 2000 },
  
  // 감상이 끝난 후 다시 대화창을 표시합니다.  
  { type: 'ui', name: 'dialogue', action: 'show', duration: 500 },
  
  { type: 'dialogue', text: '한참 동안 넋을 잃고 바라보고 말았습니다.' }
]
```

### 3.2. 오버레이 요소의 일괄 제어
특정 카테고리의 오버레이 텍스트들을 일괄적으로 소거할 때도 유용하게 사용하실 수 있습니다.  

```ts
[
  { type: 'ui', name: 'overlay-text', action: 'hide', duration: 1000 }
]
```

---

## 4. 주의 사항 (Edge Cases)

*   **상태 보존 원칙**: `ui` 커맨드를 통해 숨겨진 모듈은 내부 데이터와 현재의 스키마 상태를 완벽히 유지합니다.  다시 `show`를 호출할 경우, 소거 직전의 텍스트나 설정 그대로 화면에 복구됩니다.  
*   **명령 우선순위**: 애니메이션(`duration`)이 진행 중인 모듈에 대해 새로운 `ui` 명령이 전달되면, 기존 연출은 즉시 중단되고 최신 명령의 궤적을 따라 화면 상태가 갱신됩니다.  

---

## ⚙️ 동작 메커니즘 (Internal Mechanism)

`ui` 커맨드는 엔진의 중앙 UI 레지스트리를 참조하여 대상 모듈의 뷰(View) 인터페이스를 조절합니다.  

*   **메서드 직접 호출**: 본 커맨드는 해당 모듈의 `defineView`에서 정의된 **`show()`** 및 **`hide()`** 메서드를 직접 실행하도록 설계되었습니다.  
*   **확장성**: 사용자께서 직접 제작하신 커스텀 모듈이 적절한 가시성 제어 로직을 포함하고 있다면, `ui` 커맨드를 통해 시나리오 상에서 동일하게 제어하실 수 있습니다.  자세한 구현 방법은 [커스텀 모듈 제작 가이드](../modules.md#3-ui-렌더러-view) 문서를 참조해 주십시오.
