# 🖥️ UI Control Module (`ui`)

---

## 1. 개요 (Overview)

`ui` 모듈은 엔진에 등록된 다른 UI 모듈(대화창, 오버레이, 선택지 등)의 가시성을 직접 제어합니다. 특정 연출 도중 대화창을 잠시 숨기거나, 숨겨져 있던 UI를 강제로 다시 표시할 때 사용됩니다.

---

## 2. 커맨드 상세 (Command Reference)

### UI 제어 명령 (`UICmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'ui'` | ✅ | 커맨드 타입을 명시합니다. |
| `name` | `string` | ✅ | 조작할 모듈의 키 이름입니다. (`novel.config.ts`의 `modules`에 등록된 키) |
| `action` | `'show' \| 'hide'` | ✅ | 모듈을 보이게 할지(`show`), 숨길지(`hide`) 결정합니다. |
| `duration` | `number` | - | 페이드 인/아웃 등 전환 애니메이션의 지속 시간(ms)입니다. |

---

## 3. 활용 예시 (Examples)

### 3.1. 연출을 위한 대화창 일시 숨김
캐릭터의 전신 CG나 중요한 배경 연출을 보여주기 위해 대화창을 숨깁니다.

```ts
[
  { type: 'dialogue', text: '여기는 정말 아름다운 곳이군요.' },
  
  // 대화창 숨기기
  { type: 'ui', name: 'dialogue', action: 'hide', duration: 500 },
  
  { type: 'wait', duration: 2000 },
  
  // 다시 대화창 표시
  { type: 'ui', name: 'dialogue', action: 'show', duration: 500 },
  
  { type: 'dialogue', text: '한참을 멍하니 바라보고 있었습니다.' }
]
```

### 3.2. 오버레이 일괄 제거
이름이 지정된 여러 오버레이를 한꺼번에 숨길 때 유용합니다.

```ts
[
  { type: 'ui', name: 'overlay-text', action: 'hide', duration: 1000 }
]
```

---

## 4. 주의 사항 (Edge Cases)

*   **상태 보존**: `ui` 커맨드로 숨겨진 모듈은 내부 데이터(스키마)를 유지합니다. 다시 `show`를 호출하면 숨겨지기 전의 상태 그대로 다시 나타납니다.
*   **애니메이션 중첩**: `duration`이 진행되는 동안 동일한 모듈에 대해 다른 `ui` 커맨드를 내리면 이전 애니메이션은 중단되고 새로운 명령이 우선 적용됩니다.

---

## ⚙️ 작동 원리 (Mechanism)

`ui` 커맨드는 엔진의 내부 UI 레지스트리에서 대상 모듈을 찾아, 해당 모듈이 `defineView`에서 정의한 **`show()`** 및 **`hide()`** 메서드를 직접 실행합니다.

*   **상호작용**: 대상 모듈이 페이드 인/아웃 등의 연출 로직을 가지고 있다면, `ui` 커맨드를 통해 그 연출을 시나리오상에서 자유롭게 트리거할 수 있습니다.
*   **개발자 참고**: 커스텀 모듈을 제작할 때 이 커맨드에 대응하는 메서드를 구현하는 방법은 [커스텀 모듈 제작 가이드](../modules.md#3-ui-렌더러-view)를 참조하십시오.
