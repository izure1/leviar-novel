# 🎮 Control Module

---

## 1. 개요 (Overview)

`control` 모듈은 게임 플레이 중 사용자의 입력을 일시적으로 제한하거나 특정 시스템 동작을 제어하는 기능을 제공합니다. 주로 연출 도중 플레이어가 클릭을 통해 흐름을 끊는 것을 방지하거나, 특정 시간 동안 강제로 대기하는 연출을 구현할 때 사용됩니다.

### 주요 특징
*   **입력 및 진행 차단**: 지정된 시간 동안 마우스 클릭, 키보드 입력뿐만 아니라 엔진의 **스킵(Skip/빠른 감기)** 기능까지 일시적으로 중단시킵니다.
*   **강제 연출 대기**: 중요한 연출 구간에서 플레이어가 실수로 대사를 넘기거나 스킵으로 연출을 건너뛰는 것을 방지합니다.

---

## 2. 핵심 예제 (Main Example)

### 입력 비활성화 및 대기

```ts
[
  { type: 'dialogue', text: '잠시 후 폭발이 일어납니다. 대피하세요!' },
  
  // 2초(2000ms) 동안 입력을 막고 대기합니다.
  { type: 'control', action: 'disable', duration: 2000 },
  
  { type: 'dialogue', text: '무사히 대피했군요.' }
]
```

---

## 3. 커맨드 상세 (Command Reference)

### Control 명령 (`ControlCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'control'` | ✅ | 제어 커맨드임을 명시합니다. |
| `action` | `'disable'` | ✅ | 수행할 제어 액션입니다. 현재는 입력을 차단하는 `'disable'`만 지원합니다. |
| `duration` | `number` | ✅ | 입력을 차단하고 대기할 시간(ms)입니다. |

---

## 4. 주의 사항 (Edge Cases)

*   **진행 차단**: `duration`이 종료되기 전까지는 사용자가 클릭을 해도 다음 커맨드로 넘어가지 않습니다. 
*   **병렬 연출**: `control` 커맨드 자체는 블로킹(Blocking) 방식으로 동작하므로, 이펙트나 배경 전환과 동시에 사용하려면 `control` 커맨드 이전에 해당 연출 커맨드들을 배치해야 합니다.
*   **스킵(Skip) 모드 대응**: 플레이어가 스킵(빠른 감기) 기능을 활성화한 상태에서도 `control` 커맨드는 무시되지 않습니다. 엔진은 `duration`이 종료될 때까지 다음 커맨드로 넘어가지 않고 해당 지점에서 대기합니다.
*   **중첩 호출**: 이미 입력이 차단된 상태에서 다시 호출될 경우, 기존 남은 시간과 새로운 시간 중 더 긴 시간을 기준으로 만료 시점이 갱신됩니다.
