# 🎮 Control Module

본 문서는 `control` 모듈을 사용하여 게임 플레이 중 사용자의 입력을 정밀하게 제한하고 시스템의 실행 흐름을 제어하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`control` 모듈은 연출의 몰입도를 보호하기 위해 플레이어의 입력을 일시적으로 통제하는 기능을 수행합니다.  
긴박한 연출 도중 플레이어가 무의식적으로 클릭하여 흐름을 끊는 것을 방지하거나, 특정 시간 동안 강제적인 대기 시간을 부여하여 상황의 무게감을 전달할 때 주로 활용됩니다.  

### 주요 특징
*   **전방위적 입력 차단**: 지정된 시간 동안 마우스 클릭과 키보드 입력은 물론, 엔진의 **스킵(Skip/빠른 감기)** 기능까지 일시적으로 중단시켜 연출의 무결성을 유지합니다.  
*   **강제적 시나리오 대기**: 핵심적인 연출 구간에서 플레이어가 대사를 임의로 넘기거나 자동 진행으로 연출을 건너뛰는 행위를 효과적으로 방지합니다.  

---

## 2. 핵심 예제 (Main Example)

### 입력을 일시 비활성화하고 연출 대기 시간 부여
중요한 사건이 발생하기 직전, 사용자의 입력을 막아 긴장감을 조성하는 예시입니다.  

```ts
[
  { type: 'dialogue', text: '잠시 후 거대한 폭발이 예상됩니다.  신속히 대피하십시오!' },
  
  // 2초(2000ms) 동안 모든 사용자의 입력을 차단하고 시나리오를 대기시킵니다.  
  { type: 'control', action: 'disable', duration: 2000 },
  
  { type: 'dialogue', text: '천만다행으로 큰 피해는 면한 것 같군요.' }
]
```

---

## 3. 커맨드 상세 (Command Reference)

### Control 명령 (`ControlCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'control'` | ✅ | 시스템 제어 커맨드임을 명시합니다. |
| `action` | `'disable'` | ✅ | 수행할 제어 동작입니다.  현재는 입력 차단 기능인 `'disable'`을 지원합니다. |
| `duration` | `number` | ✅ | 입력을 차단하고 시나리오를 정지시킬 총 시간(ms)입니다. |

---

## 4. 주의 사항 (Edge Cases)

*   **실행 흐름의 정지**: `duration`으로 지정된 시간이 경과하기 전까지는 사용자가 어떠한 상호작용을 시도하더라도 다음 커맨드로 이행되지 않습니다.  
*   **연출의 배치 순서**: `control` 커맨드는 시나리오의 진행을 멈추는 블로킹(Blocking) 방식으로 작동합니다.  따라서 배경 전환이나 물리 효과와 동시에 연출을 진행하시려면, 반드시 `control` 커맨드 이전에 해당 연출 명령들을 배치해 주시기 바랍니다.  
*   **스킵(Skip) 모드와의 상호작용**: 플레이어가 빠른 감기(Skip) 기능을 활성화한 상태라 하더라도, `control` 커맨드는 생략되지 않습니다.  엔진은 지정된 시간이 종료될 때까지 해당 지점에서 엄격하게 대기합니다.  
*   **중첩 호출 시의 정책**: 이미 입력이 차단된 상태에서 새로운 `control` 명령이 전달될 경우, 기존의 잔여 시간과 새로운 대기 시간 중 더 긴 시간을 기준으로 해제 시점이 갱신됩니다.
