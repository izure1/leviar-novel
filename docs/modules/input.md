# ⌨️ Input Module

본 문서는 `input` 모듈을 사용하여 사용자로부터 텍스트 정보를 수집하고 이를 엔진의 변수에 저장하는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`input` 모듈은 플레이어와의 직접적인 소통을 위한 인터페이스를 제공합니다.  
주인공의 성명을 결정하거나 암호를 입력받는 등, 사용자의 텍스트 데이터를 시나리오 로직에 반영해야 하는 상호작용 상황에서 필수적으로 활용됩니다.  

### 주요 특징
*   **지능형 변수 할당**: 입력된 텍스트 데이터를 지정된 변수 식별자(`to`)에 자동으로 저장합니다.  `_` 접두사 규칙에 따라 전역 또는 지역 스코프가 결정됩니다.  
*   **다양한 입력 모드**: 한 줄 단위의 간결한 입력부터 장문의 일기 형식을 위한 멀티라인(`textarea`) 모드까지 폭넓게 지원합니다.  
*   **커스터마이징 가능한 버튼**: 확인 및 취소 버튼을 자유롭게 구성할 수 있으며, 각 버튼에 고유한 시각적 스타일과 로직을 부여할 수 있습니다.  
*   **안정적인 포커스 유지**: 전체화면이나 모바일 기기 등 다양한 실행 환경에서도 입력 포커스가 유실되지 않도록 정밀하게 설계되었습니다.  

---

## 2. 핵심 예제 (Main Example)

### 성명 입력 및 기록 작성 구현

```ts
[
  // 1. 기본적인 이름 입력 (엔터 키를 통한 신속한 완료를 지원합니다)
  { 
    type: 'input', 
    to: 'playerName', 
    label: '귀하의 존함을 입력해 주십시오.' 
  },

  // 2. 다중 라인 메모 입력 및 취소 로직 구성
  {
    type: 'input',
    to: '_currentMemo', // 현재 씬 내에서만 유효한 지역 변수에 저장합니다.  
    label: '오늘의 회고를 작성해 주십시오.',
    multiline: true,
    buttons: [
      { text: '저장 및 진행' },
      { text: '작성 취소', cancel: true } // 데이터를 보존하지 않고 입력을 종료합니다.  
    ]
  }
]
```

## 3. 커맨드 상세 (Command Reference)

### Input 명령 (`InputCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'input'` | ✅ | 입력 커맨드임을 명시합니다. |
| `to` | `string` | ✅ | 입력된 값을 할당할 변수의 이름입니다. |
| `label` | `string` | - | 입력창 상단에 표시되어 사용자를 안내하는 문구입니다. |
| `multiline` | `boolean` | - | `true` 설정 시 장문 입력을 위한 여러 줄 모드로 전환됩니다. |
| `buttons` | `InputButton[]` | - | 하단에 배치할 버튼들의 구성 목록입니다. |

#### InputButton 객체 상세 명세

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `text` | `string` | ✅ | 버튼 표면에 표시될 텍스트입니다. |
| `cancel` | `boolean` | - | `true` 설정 시 변수 업데이트 없이 입력 시퀀스를 종료합니다. |
| `style` | `object` | - | 버튼의 배경 및 형태에 대한 커스텀 스타일입니다. |
| `textStyle` | `object` | - | 버튼 내부 텍스트의 폰트 및 색상 스타일입니다. |

## 5. 동작 메커니즘 (Internal Logic)

### 포커스 및 렌더링 동기화
*   **강제 포커스 시스템**: 입력 모듈이 활성화된 동안 엔진은 보이지 않는 DOM 요소를 통해 사용자의 입력을 수집하며, 지속적으로 포커스를 복구하여 입력의 끊김을 방지합니다.  
*   **캔버스 동기화**: 실제 입력은 시스템 레벨에서 처리되나, 화면상에는 엔진의 렌더링 시스템을 통해 정교하게 디자인된 텍스트와 커서가 실시간으로 동기화되어 표시됩니다.  

---

## 6. 주의 사항 (Edge Cases)

*   **엔터 키 처리 정책**: `multiline: false` 모드에서 엔터 키 입력 시, 버튼 목록 중 첫 번째(인덱스 0) 항목을 클릭한 것과 동일한 절차를 수행합니다.  
*   **취소(Cancel)의 영향**: `cancel: true`가 설정된 버튼을 통해 종료할 경우, 입력된 데이터는 변수에 저장되지 않으며 기존의 변수 상태가 그대로 유지됩니다.  
*   **입력 무결성**: 입력이 완료되어 변수에 저장된 데이터는 엔진의 영속성 시스템에 의해 세이브 데이터의 일부로 관리됩니다.
