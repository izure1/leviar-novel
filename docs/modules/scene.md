# 🎬 Scene Module

---

## 1. 개요 (Overview)

`scene` 모듈은 현재 씬의 실행을 멈추고 다른 씬을 **서브루틴(Subroutine)**처럼 호출하는 기능을 제공합니다. 

호출된 서브씬이 모두 종료되면(더 이상 진행할 커맨드가 없으면) 자동으로 호출자(Caller) 씬의 원래 위치로 되돌아옵니다. 옵션을 통해 호출 시점의 화면, 오디오, 데이터를 완전히 보존하고 복구할 수 있어 UI 화면이나 공통 연출을 구현하는 데 필수적입니다.

---

## 2. 핵심 예제 (Main Example)

```ts
[
  { type: 'dialogue', text: '인벤토리를 엽니다.' },

  // 인벤토리 씬을 호출. 화면 배경을 유지하고 복귀 시 상태 복구.
  { 
    type: 'scene', 
    call: 'scene-inventory', 
    preserve: true, 
    restore: true 
  },

  { type: 'dialogue', text: '인벤토리를 닫았습니다.' }
]
```

---

## 3. 커맨드 상세 (Command Reference)

### Scene 호출 명령 (`SceneCallCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'scene'` | ✅ | 씬 호출 커맨드임을 명시합니다. |
| `call` | `string` | ✅ | 호출할 대상 서브씬의 이름입니다. `novel.config.ts`에 등록된 씬 이름이어야 합니다. |
| `preserve` | `boolean` | | **`true`**: 현재 화면/오디오 상태를 유지한 채 서브씬을 시작합니다. 이때 호출자의 모듈 `state`가 서브씬의 `initial` 설정을 완전히 대체(Replace)하여 그대로 상속됩니다.<br>**`false`** (기본값): 화면/오디오를 초기화한 후 백지 상태에서 서브씬의 `initial`만 적용하여 시작합니다. |
| `restore` | `boolean` | | **`true`**: 서브씬 종료 후 호출 시점의 화면/오디오 상태로 완벽 복구합니다.<br>**`false`** (기본값): 서브씬의 화면 상태를 이어받으며 변수/진행 커서만 복구합니다.<br>*(참고: `preserve: false`일 경우 무조건 `true`처럼 동작합니다.)* |

---

### 💡 `preserve: true` 일 때의 상태(State) 관리
서브씬 진입 시 `preserve: true`를 사용하면 모듈들의 상태(`stateStore`)가 초기화되지 않고 서브씬으로 상속됩니다.
이때 **서브씬 정의(`defineScene`)에 작성된 `initial` 값은 무시되며, 상속받은 상태 객체가 이를 완전히 대체(Replace)**하여 사용됩니다.
예를 들어, 호출자 씬에 띄워둔 캐릭터와 배경, 대화창 스타일 등은 어떠한 변경도 없이 완벽하게 유지(동결)된 상태로 서브씬 스크립트가 시작됩니다.

---

## 4. 주의 사항 (Edge Cases)

> [!WARNING]
> **서브씬 내부에서 상위 씬(호출자)으로 강제 전환(`next`)하지 마세요!**
> 중첩 씬은 함수 콜 스택(Call Stack) 방식으로 관리되며, 다른 씬으로 이동(`next`)해도 스택은 자동으로 폐기되지 않습니다. 따라서 서브씬에서 호출자나 그 이상의 상위 씬으로 강제 이동하게 되면, 복귀를 기다리던 기존 상위 씬들이 영원히 메모리에 남게 되어 **메모리 누수**가 발생합니다. 서브씬이 끝난 뒤에는 반드시 자연스럽게 배열의 끝에 도달하여 복귀(Return)되도록 설계해야 합니다.

* **재귀 및 다중 스택 보장**: 서브씬 내에서 또 다른 서브씬을 호출할 수 있습니다. 내부적인 콜 스택(Call Stack)을 통해 무한한 깊이의 중첩 호출도 안전하게 보장됩니다.
