# 🎬 Scene Module

본 문서는 `scene` 모듈을 사용하여 현재 씬의 실행을 일시 중단하고 다른 씬을 **서브루틴(Subroutine)**처럼 호출 및 복귀시키는 방법을 기술합니다.  

---

## 1. 개요 (Overview)

`scene` 모듈은 시나리오의 구조적 설계를 지원하는 핵심 도구입니다.  
호출된 서브씬의 모든 커맨드가 종료되면, 엔진은 자동으로 호출자(Caller) 씬의 원래 위치로 돌아와 다음 명령을 수행합니다.  
이 과정에서 호출 시점의 화면, 오디오, 데이터 상태를 정밀하게 보존하고 복구할 수 있어, 시스템 UI나 공통 연출을 모듈화하여 관리하는 데 필수적인 기능을 제공합니다.  

---

## 2. 핵심 예제 (Main Example)

### 중첩 씬 호출 및 복귀 구현
인벤토리와 같은 시스템 화면을 호출한 뒤, 사용자 작업이 끝나면 원래의 시나리오 지점으로 매끄럽게 복귀하는 예시입니다.  

```ts
[
  { type: 'dialogue', text: '인벤토리를 활성화합니다.' },

  // 인벤토리 서브씬을 호출합니다.  
  // 호출 시점의 화면과 오디오 상태를 유지(preserve)하고 종료 시 완벽히 복구(restore)합니다.  
  { 
    type: 'scene', 
    call: 'scene-inventory', 
    preserve: true, 
    restore: true 
  },

  { type: 'dialogue', text: '인벤토리 확인을 완료하고 시나리오로 돌아왔습니다.' }
]
```

## 3. 커맨드 상세 (Command Reference)

### Scene 호출 명령 (`SceneCallCmd`)

| 속성 | 타입 | 필수 | 설명 |
| :--- | :--- | :---: | :--- |
| `type` | `'scene'` | ✅ | 씬 호출 커맨드임을 명시합니다. |
| `call` | `string` | ✅ | 호출할 대상 서브씬의 식별자입니다.  (`config`에 등록된 이름) |
| `preserve` | `boolean` | - | **true**: 현재 화면과 오디오 상태를 유지한 채 서브씬을 시작합니다.  이때 호출자의 모듈 상태(`state`)가 서브씬의 초기 설정(`initial`)을 대체하여 상속됩니다.<br>**false** (기본값): 화면을 백지 상태로 초기화한 후 서브씬을 시작합니다.  단, 오디오는 전역 요소이므로 중단 없이 재생이 유지됩니다. |
| `restore` | `boolean` | - | **true**: 서브씬 종료 후 호출 시점의 시각적/청각적 상태로 완벽히 복구합니다.<br>**false** (기본값): 서브씬의 화면 상태를 그대로 이어받으며, 변수와 진행 커서만 호출자 시점으로 복구합니다. |

---

## 4. 고급 상태 관리 안내

### `preserve: true` 적용 시의 작동 원리
서브씬 진입 시 이 옵션을 활성화하면 엔진의 상태 저장소(`stateStore`)가 초기화되지 않고 서브씬으로 안전하게 상속됩니다.  
이때 **서브씬 정의(`defineScene`)에 포함된 `initial` 설정은 무시되며, 상속받은 실제 데이터가 이를 완전히 대체**합니다.  
이를 통해 호출자 씬의 캐릭터 배치, 배경 연출, 대화창 스타일 등이 어떠한 시각적 끊김 없이 완벽하게 유지된 상태에서 서브씬 스크립트를 실행할 수 있습니다.  

---

## 5. 주의 사항 (Edge Cases)

> [!WARNING]
> **서브씬 내부에서 상위 씬으로의 강제 전환(`next`)을 지양하십시오.**  
> 중첩 씬은 프로그래밍의 함수 콜 스택(Call Stack) 방식으로 정밀하게 관리됩니다.  서브씬에서 상위 씬으로 `next`를 통해 강제 이동할 경우, 복귀를 대기하던 상위 씬 정보가 메모리에 영구적으로 남게 되어 **메모리 누수**의 원인이 됩니다.  서브씬의 시나리오가 끝난 후에는 반드시 자연스럽게 종료(Return)되어 시스템이 스택을 소거할 수 있도록 설계해 주시기 바랍니다.  

*   **재귀적 호출 보장**: 서브씬 내에서 또 다른 서브씬을 호출하는 중첩 구조를 완벽하게 지원합니다.  내부 콜 스택 시스템을 통해 안정적인 다중 계층 호출이 가능합니다.  
*   **오디오 복구 정책**: `restore: true` 적용 시, 서브씬에서 다른 음악이 재생되었다면 호출 시점의 원래 곡과 재생 위치로 정밀하게 되감아 복구합니다.  동일한 곡이 유지되었다면 어떠한 끊김도 없이 재생이 지속됩니다.
