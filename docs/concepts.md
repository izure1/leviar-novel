## <a id="overview"></a>1. 핵심 메커니즘 개요 (Overview)

| 개념 | 설명 | 비고 |
| :--- | :--- | :--- |
| **Resolvable** | 모든 속성에 동적 함수 사용 허용 | 시나리오 흐름 내 조건부 분기 로직 최소화 |
| **Skip & Parallel** | 병렬 연출 및 쾌속 진행 시스템 | 입체적인 동시 연출 및 사용자 편의성 제공 |
| **Persistence** | 변수와 모듈 상태의 통합 관리 | 복잡한 UI 상태까지 보존하는 완벽한 세이브/로드 |
| **Track & Clear** | 자동화된 객체 생명주기 관리 | 메모리 누수 방지 및 간결한 씬 전환 로직 |

### 스킵 메커니즘: 병렬 연출과 빠른 감기
엔진에는 두 가지 종류의 '스킵'이 존재합니다.

#### **A. 병렬 연출 (`skip: true` 속성)**
명령어 단위의 설정입니다. 해당 연출을 실행하되 엔진의 실행 포인터는 즉시 다음 커맨드로 넘깁니다. 배경 전환 중에 대사를 출력하거나, 여러 이미지 효과를 동시에 적용할 때 사용합니다.

#### **B. 빠른 감기 (`novel.skip()` 메서드)**
사용자가 이미 본 내용을 빠르게 넘길 때 사용합니다. 이 모드에서는 `Renderer`가 애니메이션의 지속 시간(`duration`)을 강제로 **0**으로 처리하여 모든 연출이 즉시 완료되도록 합니다.

> [!TIP]
> 모듈 개발 시 애니메이션 시간에 `ctx.renderer.dur(1000)`를 사용하면, 엔진의 빠른 감기 상태에 맞춰 시간이 자동으로 0으로 조정됩니다.

### 자원 관리와 객체 추적 (`Track & Clear`)
캔버스 기반 렌더링에서 가장 흔한 실수는 씬 전환 시 이전의 객체를 삭제하지 않아 발생하는 메모리 누수입니다.
* **`Renderer.track(obj)`**: 모듈이 생성한 모든 오브젝트를 엔진의 추적 목록에 등록합니다.
* **`Renderer.clear()`**: 씬이 종료되거나 전환될 때, 엔진은 추적 중인 모든 객체의 `remove()` 메서드를 호출하여 화면과 메모리에서 안전하게 소거합니다.

## <a id="scenes"></a>2. 장면 시스템 (Scenes)

`defineScene` 함수를 통해 독립적인 이야기의 단위인 씬을 구성합니다. 씬은 고유의 로컬 상태와 초기 UI 환경을 가집니다.

| 속성명 | 타입 | 도입 이유 (Why) |
| :--- | :--- | :--- |
| **`variables`** | `object` | 씬 내 전용 임시 상태 관리 (캡슐화 및 메모리 격리) |
| **`initial`** | `object` | 씬 로드 즉시 **모듈의 Scheme State를 강제 초기화** |
| **`next`** | `string | object` | 시나리오 종료 후 자동 이동 경로 지정 |
| **`hooks`** | `HookDescriptor` | 씬 전용 라이프사이클 훅 등록 |

### 초기 상태 설정 (`initial`) — 무대 뒤의 준비
`initial`은 모듈이 씬에서 활성화되는 **탄생 시점(Initialization)**에 외부에서 해당 모듈의 초기 성격이나 시각적 환경을 결정하는 데이터를 전달하는 역할을 합니다.

| 구분 | 비유 | 작동 시점 | 개입 깊이 |
| :--- | :--- | :--- | :--- |
| **초기 설정 (`initial`)** | **분장 및 의상 (Backstage)** | 막이 오르기 전 | **심층적**: 배우의 외형 자체를 바꿈 |
| **명령어 (`Command`)** | **실시간 연기 지시 (On-stage)** | 연극 진행 중 | **외부적**: 배우에게 행동을 시킴 |

## <a id="variables"></a>3. 변수 시스템 (Variables)

`fumika` 엔진의 변수 시스템은 **명명 규칙(Naming Convention)을 통해 런타임 스코프를 결정**합니다.

| 항목 | 전역 변수 (Global) | 지역 변수 (Local) |
| :--- | :--- | :--- |
| **명명 규칙** | **접두사 없음** (예: `gold`) | **`_` 접두사 필수** (예: `_temp`) |
| **생명주기** | 게임 전체 유지 | 해당 씬 한정 (전환 시 초기화) |
| **판별 로직** | `!name.startsWith('_')` | `name.startsWith('_')` |

> [!CAUTION]
> 접두사 규칙은 단순한 스타일 가이드가 아닌 **기능적 스위치**입니다. 규칙을 어기면 엔진이 잘못된 스토어를 참조하여 데이터 유실이 발생할 수 있습니다.

## <a id="resolvable"></a>4. 동적 속성 (Resolvable)

`Resolvable`은 많은 커맨드 속성들이 고정된 값뿐만 아니라 **함수**를 허용하게 하는 디자인 패턴입니다.

### 기본 동작 원리
속성값으로 함수가 전달되면, 엔진은 해당 커맨드가 실행되는 시점에 함수를 호출하고 그 반환값을 실제 값으로 사용합니다.

```ts
// 함수형 값 (Resolvable) 사용 예시
text: (vars) => `안녕하세요, ${vars.player_name}님!`
```

## <a id="hooks"></a>5. 훅 시스템 (Hooks)

프레임워크 내부의 데이터 흐름에 개입하여 값을 변경하거나 특정 시점에 로직을 실행하는 이벤트 인터셉터 메커니즘입니다.

### 작동 원리: 릴레이(Relay) 방식
여러 개의 훅이 등록되어 있을 때, 엔진은 이를 **릴레이 경주**처럼 처리합니다. 첫 번째 훅이 수정한 결과값이 두 번째 훅의 입력값으로 전달됩니다.

### 엔진 전역 훅 (`novel:`)
* `novel:scene`: 새로운 씬이 로드될 때
* `novel:next`: 다음 단계로 넘어갈 때 (진행 차단 가능)
* `novel:save` / `novel:load`: 세이브/로드 시점

## <a id="debug-mode"></a>6. 디버그 모드 (Debug Mode)

UI 요소의 **실제 영역(Hitbox)과 레이아웃 경계**를 시각화하여 정확한 위치 파악을 돕는 도구입니다.

* **활성화**: `novel.debugMode = true`
* **표시 항목**: 영역 외곽선(Outlines), 숨겨진 레이아웃 가이드 등
