# 🔄 Resolvable (함수형 값)

---

## 1. 개요 (Overview)

`Resolvable`은 `fumika` 엔진 전반에서 사용되는 핵심 디자인 패턴입니다. 많은 커맨드 속성들이 고정된 값뿐만 아니라 **함수**를 허용하며, 이를 통해 런타임의 변수 상태에 따라 동적으로 변하는 연출을 손쉽게 구현할 수 있습니다.

### 주요 이점
* **동적 연출**: 현재 변수 상태(`Vars`)에 따라 텍스트나 동작을 실시간으로 결정합니다.
* **상태 참조**: 기존 값을 참조하여 새로운 값을 계산하는 연산(`+=`, `-=` 등)을 지원합니다.
* **유연한 제어**: 복잡한 조건부 로직을 씬 스크립트 내에서 함수 형태로 직접 작성할 수 있습니다.

---

## 2. 기본 동작 원리 (How it Works)

속성값으로 함수가 전달되면, 엔진은 해당 커맨드가 실행되는 시점에 함수를 호출하고 그 반환값을 실제 값으로 사용합니다.

### 함수 시그니처
대부분의 Resolvable 함수는 현재 활성화된 변수 객체(`vars`)를 인자로 받습니다.

```ts
// 1. 단순 값 (Static)
text: '안녕하세요!'

// 2. 함수형 값 (Resolvable)
text: (vars) => `안녕하세요, ${vars.player_name}님!`
```

---

## 3. 주요 활용 사례 (Use Cases)

### 3.1. 대사 및 텍스트 (`dialogue`, `choices`)
변수 값을 문자열 내에 삽입하거나 상황에 맞는 문구를 출력합니다.

```ts
// 대사 예시
{ type: 'dialogue', text: ({ gold }) => `현재 잔액은 ${gold}G 입니다.` }

// 선택지 예시
{ 
  text: ({ heart }) => heart > 50 ? '고백한다' : '지켜본다',
  goto: ({ heart }) => heart > 50 ? 'confess' : 'wait'
}
```

### 3.2. 변수 조작 (`var`)
기존 값을 바탕으로 변수 값을 갱신합니다.

```ts
// 기존 호감도(heart)에 10을 가산
var: ({ heart }) => ({ heart: heart + 10 })

// 조건부 변수 조작
var: ({ gold, is_vip }) => ({ 
  gold: is_vip ? gold - 50 : gold - 100 
})
```
> [!TIP]
> 전역 변수와 지역 변수의 구분 등 변수 시스템의 기초는 [Variables 가이드](./variables.md)를 참조하십시오.

### 3.3. 조건 분기 (`condition`)
특정 조건의 충족 여부를 판단하여 시나리오 흐름을 제어합니다.

```ts
{ 
  type: 'condition', 
  if: ({ gold }) => gold >= 1000, 
  goto: 'rich_route' 
}
```

---

## 4. 주의 사항 (Edge Cases)

* **순수성 유지**: Resolvable 함수 내부에서 외부 변수를 직접 수정하기보다는, 반드시 반환값(Return)을 통해 결과를 전달해야 합니다.
* **동기 실행**: Resolvable 함수는 동기적으로 실행되어야 하며, `Promise`를 반환하는 비동기 함수는 지원되지 않습니다.
* **성능**: 텍스트 Resolvable은 화면이 렌더링될 때마다 호출될 수 있으므로, 내부에서 너무 무거운 연산을 수행하지 않도록 주의하십시오.
