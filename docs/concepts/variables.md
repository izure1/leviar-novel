# 💎 Variables (변수 시스템)

---

## 1. 개요 (Overview)

`fumika` 엔진의 변수 시스템은 게임의 상태를 보존하고 시나리오의 흐름을 결정하는 핵심 인프라입니다. 본 엔진은 **명명 규칙(Naming Convention)을 통해 런타임 스코프를 결정**하는 방식을 채택하고 있습니다. 이는 정적 타입 체크뿐만 아니라 엔진의 동적 로직 실행 시 변수의 저장 위치와 생명주기를 판별하는 절대적인 기준이 됩니다.

## 2. 변수 유형 및 스코프 (Variable Types)

| 항목 | 전역 변수 (Global) | 지역 변수 (Local) |
| :--- | :--- | :--- |
| **명명 규칙** | **접두사 없음** (예: `gold`) | **`_` 접두사 필수** (예: `_temp`) |
| **정의 위치** | `novel.config.ts`의 `variables` | `defineScene`의 `variables` |
| **생명주기** | 게임 전체 (씬 전환 시 유지) | 해당 씬 한정 (씬 전환 시 초기화) |
| **저장 위치** | `Novel.variables` (Global Store) | `Scene.localVars` (Instance Store) |
| **판별 로직** | `!name.startsWith('_')` | `name.startsWith('_')` |

### 2.1. 전역 변수 (Global Variables)
게임 전체의 영속적인 데이터(경험치, 플래그, 플레이어 이름 등)를 관리합니다. 모든 씬에서 공유되며 세이브 데이터의 핵심을 이룹니다.

### 2.2. 지역 변수 (Local Variables)
장면 내의 일시적인 상태(특정 구간의 선택 횟수, 장면 전용 연출 플래그 등)를 관리합니다. **씬 전환 시 `preserve` 옵션의 사용 여부와 무관하게 항상 초기화**되어 메모리 효율성을 극대화합니다.

---

## 3. 명명 규칙의 엄격성 (Naming Strictness)

### 3.1. 런타임 스코프 판별 (Runtime Scoping)
엔진은 실행 중에 변수명의 첫 글자가 `_`인지를 확인하여 데이터를 처리할 위치를 결정합니다.

*   **`_`로 시작**: 지역 변수 스토어(`localVars`)에 접근.
*   **`_` 이외의 문자**: 전역 변수 스토어(`globalVars`)에 접근.

> [!CAUTION]
> **런타임 주의 사항 (Runtime Warning)**
> 접두사 규칙은 단순한 스타일 가이드가 아닌 **기능적 스위치**입니다. 타입 체크(TypeScript)가 보장하더라도, 런타임에서 규칙을 어기고 전역 변수를 `_`로 시작하게 하거나 지역 변수에서 `_`를 누락하면 엔진이 잘못된 스토어를 참조하게 됩니다. 이로 인해 데이터 유실이나 의도치 않은 변수 덮어쓰기가 발생할 수 있으므로 절대적으로 준수해야 합니다.

---

## 4. 생명주기 및 초기화 (Lifecycle)

### 4.1. 씬 전환 (Transition)
씬이 전환되거나 같은 씬으로 다시 진입(`loadScene`)하는 경우:
1.  **전역 변수**: 기존의 값을 그대로 유지합니다.
2.  **지역 변수**: 현재 씬 인스턴스가 파괴되고 새 인스턴스가 생성되므로, **정의된 초깃값으로 강제 리셋**됩니다.

### 4.2. `preserve` 옵션의 범위
`preserve: true` 옵션은 렌더링 자원(배경, 캐릭터 등)과 모듈의 뷰 상태만 유지합니다. **지역 변수의 보존과는 무관**하며, 로직상의 데이터 유지가 필요하다면 반드시 전역 변수 스코프를 활용해야 합니다.

---

## 5. 변수 조작 및 활용 (Usage)

### 5.1. `var` 커맨드를 통한 동적 할당
정적 값뿐만 아니라 함수(Resolvable)를 전달하여 현재 상태를 기반으로 한 연산이 가능합니다.

```ts
// 전역 변수 차감 및 지역 변수 가산 예시
{
  type: 'var',
  name: 'gold',
  value: ({ gold }) => gold - 100
},
{
  type: 'var',
  name: '_stepCount',
  value: ({ _stepCount }) => _stepCount + 1
}
```

### 5.2. 텍스트 보간 (Interpolation)
대사 텍스트 내에서 `{{ expr }}` 형식을 사용하여 실시간 변수 값을 출력합니다.

```ts
{ 
  type: 'dialogue', 
  text: '현재 소지금: {{gold}}G / 진행도: {{_stepCount}}단계' 
}
```

---

## 6. 결론 (Conclusion)

`fumika`의 변수 시스템은 명시적인 접두사 규칙을 통해 스코프를 강제함으로써 개발자의 실수를 미연에 방지합니다. **"전역은 접두사 없이, 지역은 `_`와 함께"**라는 대원칙을 준수할 때만 타입 안전성과 런타임 정합성이 완벽하게 보장됩니다.
