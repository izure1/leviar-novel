# 

1. 텍스트 출력 시, 이름창 부분은 transition 되어선 안되며, 즉시 바뀌어야합니다.
2. 이름창과 텍스트 창은 다른 text 객체여야 합니다.
3. 이름창과 텍스트창의 글꼴, 크기, 색상 등의 스타일을 사용자가 설정으로 지정할 수 있어야 합니다.
4. 씬은 자체적인 지역변수를 가질 수 있어야 합니다. 전역변수와 이름이 겹칠 순 없으며, 씬이 전환되면 지역변수는 초기화되어야 합니다.
5. 플레이 도중 언제나 세이브할 수 있어야 하는데, 현재 씬의 렌더러 상태를 복원할 수 있어야 합니다. 씬의 지역변수, 전역 변수, 복원 방식은 현재 씬의 cursor 위치 (dialogues 배열의 인덱스)를 기반으로 복원할 수 있어야 합니다.
6. 빠른 재생감기(skip) 기능을 지원할 것입니다. 따라서 animate 효과는 +=, -=, /=, *= 등의 복합연산을 사용해서는 안됩니다. (이유: 빠르게 감기를 할 시, 멈췄을 때 기존의 값을 복원하는 것이 어려움. 예를 들어 3초에 걸쳐 100px씩 증가하는 애니메이션을 1초만에 끝낼 시, 100px만 증가해야하는데 300px이 증가되어버릴수도 있음. 따라서 start, end 값을 미리 계산해놓고, duration 시간동안 선형적으로 변화하도록 해야함. 또한 animate 도중 세이브를 해버리면 복잡해짐.)
7. 위 5-6번 내용으로 인해 캐릭터의 카메라 포커스, 캐릭터 이동 등을 animate로 처리하더라도, 실제 좌표의 정보는 데이터로 관리되어야 합니다. (즉, 렌더링 되는 화면상의 좌표와 실제 데이터상의 좌표가 일치해야 합니다.) 그래야 세이브 후 렌더링을 다시 했을 때, 캐릭터의 위치 등이 이상해지지 않고, 빠르게 감기(skip) 중에도 부드럽게 자연스러운 연출이 가능합니다.
8. 빠른 감기 도중에는 텍스트 타이핑 효과는 일어나지 않아야 하며, 즉시 모든 텍스트가 출력되어야 합니다. animate 효과 역시 즉시 적용되어야 합니다.
